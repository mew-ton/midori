# Midori 概要

> ステータス：設計フェーズ
> 最終更新：2026-04-03

## 概要

任意の入力ソース（MIDI / OSC 等）の信号を受け取り、任意の出力先へ変換して送信するブリッジツール。演奏のリアルタイム同期（打鍵・エクスプレッション・左右ピッチ等）を主目的とし、VRChat アバター制御をプライマリターゲットとする。

ツールは **ブリッジ（Runtime）** と **GUI** に分離しており、ブリッジは設定ファイルを読んで動作するだけの自立したバイナリとして機能する。GUI はブリッジと疎結合で、モニタリングおよび設定編集を担う。

将来的には VRChat 向けに公開されているあらゆる OSC ツールの用途（心拍・顔トラッキング・キーボードホットキー・Twitch 連携等）をカバーする汎用パイプラインを目指す。

## ユースケース（初回）

- エレクトーンのアバターモデルを VRChat に持ち込み、演奏に合わせて打鍵・エクスプレッション・ペダルをリアルタイムに同期させる
- 楽器プロファイルをコミュニティで共有・配布する
- 他機種の MIDI 楽器ユーザーが自前のプロファイルを作成して利用する

## 配布モデル

ワークスペース（= git リポジトリ）単位で配布する。ファイル単体のインストールは行わない。

```
コミュニティで公開・共有されるもの

  プラグインリポジトリ  ルートに .midori/plugin.yaml を持つ git リポジトリ。
                        devices/*.yaml / ドライバー / デバイス種別定義 等を提供する。
                        → URL を GUI に貼るだけで <app-data-dir>/plugins/ にインストールされる

プライベートで受け渡しされるもの（公開を前提としない）

  ワークスペース        devices/ / mappers/ / profiles/ を含む git リポジトリ。
                        楽器プロファイル・アバター設定・マッピングを含む。
                        → リポジトリ単位でクローン・共有する

共有しないもの

  preferences.yaml      OS アプリデータディレクトリに保存。環境固有の UI・AI 設定を含む。
                        ワークスペースには含まれない
```

## セキュリティ要件

詳細設計 → [`11-security/`](11-security/)

- ブリッジは明示的に渡された設定ファイルのみを読む
- ファイルシステムアクセスは preferences で許可されたパスに限定する
- ネットワーク送信はプロファイルの connection で定義された送信先のみ
- GUI の IPC は許可チャンネルのみを expose する
- ドライバーはコミュニティ由来のバイナリであり、段階的にサンドボックス化する（`11-security/01-driver-sandbox.md`）
- プラグインが提供するウィジェット（iframe）は sandbox 属性と CSP で閉じ込める（`11-security/02-widget.md`）
- AI エージェントのツール権限はファイルパス・操作種別単位で制限する（`11-security/03-ai.md`）
