# 変換グラフ編集画面

## メタファー

| サーフェス | メタファー |
|---|---|
| 画面全体 | **ブレッドボード** — IC チップと導線で回路を組み上げる作業台 |
| デバイスブロック | **DIP パッケージ IC** — 両側にピン（ポート）が並ぶ。折りたたみはチップを伏せる操作 |
| 変換ノード | **ロジックゲート・オペアンプ** — 入力→処理→出力が一体の小さな回路素子 |
| 接続線 | **ジャンパーワイヤー** — 被覆の色 = 信号の型。物理的に「つながっている」状態 |
| ポート形状 | **IC のピン形状** — 型ごとに形を変えることで、接続可否を物理的な嵌合として表現 |
| テスター | **マルチメーター・オシロスコーププローブ** — 回路の任意の点に当てて信号を観測する |

---

デバイス構成を接続する変換グラフをノードグラフで定義する画面。インタラクションは **Blender のシェーダーエディター** を参考にする。

```
入力ブロック群（左）          計算ノード群（中央）          出力ブロック群（右）
デバイスごとに 1 ブロック                                   デバイスごとに 1 ブロック
縦に並ぶ・折りたたみ可能                                    縦に並ぶ・折りたたみ可能
```

| 要素 | 内容 |
|---|---|
| 入力ブロック群（左端） | 入力デバイス 1 つにつき 1 ブロック。ブロックのタイトルは `device_id`。縦に並ぶ |
| 計算ノード | キャンバス上に自由に配置・接続できる |
| 出力ブロック群（右端） | 出力デバイス 1 つにつき 1 ブロック。同様に縦に並ぶ |
| 接続線 | ポート間をベジエ曲線で結ぶ。スカラー型は実線、配列型は破線。型不一致の接続は不可 |

各ブロックはタイトル行クリックで折りたたみ可能。折りたたみ時はタイトルのみ表示し、接続線はポートの代わりにブロック端に集約する。

```
展開時:
┌─ ▼ yamaha-els03 ──┐                    ┌─ ▼ vrchat-default ┐
│ upper.{note}.pressed ○[]  ──────────►  │ upper_key.{note} ○[]
│ upper.{note}.vel    ◇[]                │ expression.value ◇
│ expression.value    ◇  ──────────────► └───────────────────┘
└────────────────────┘

折りたたみ時:
┌─ ► yamaha-els03 ──┐ ─────────────────► ┌─ ► vrchat-default ┐
└────────────────────┘                    └───────────────────┘
```

## ポートの型と視覚表現

型ごとに**形状**と**色**で識別する。色だけに依存しない（[→ 01-principles.md アクセシビリティ](01-principles.md)）。

### スカラー型

| 型 | 形状 | oklch |
|---|---|---|
| `bool` | ○（小円） | `oklch(62% 0.09 255)` |
| `float` | ◇（菱形） | `oklch(70% 0.10 75)` |
| `int` | □（四角） | `oklch(63% 0.09 195)` |
| `pulse` | △（三角） | `oklch(58% 0.09 305)` |

### 配列型：固定長と可変長は異なる型

**システム境界（デバイスブロックのポート）では配列は常に固定長（`T[N]`）。** 可変長配列（`T[]`）はグラフ内部のノード間でのみ出現しうる（`compact` ノードが生成する可変長配列がその例）。

**固定長配列**（`T[N]`）と**可変長配列**（`T[]`）は互換性がない別の型である。視覚的にも区別する。

| 型 | 形状 | 視覚表現 |
|---|---|---|
| `bool[N]`（固定長） | ○ の二重輪郭 | 外側の輪郭に N の数字バッジ |
| `float[N]`（固定長） | ◇ の二重輪郭 | 同上 |
| `int[N]`（固定長） | □ の二重輪郭 | 同上 |
| `bool[]`（可変長） | ○ の二重輪郭（破線） | バッジなし |
| `float[]`（可変長） | ◇ の二重輪郭（破線） | 同上 |
| `int[]`（可変長） | □ の二重輪郭（破線） | 同上 |

- **二重輪郭（実線）** = 固定長配列。サイズが定義から確定している
- **二重輪郭（破線）** = 可変長配列。サイズが実行時に決まる
- 接続線はスカラーが実線、配列型は破線（固定・可変共通）

### `{note}` と `*` の Array 表現

`{note}`（per-key 展開）は `T[N]`（固定長）、`*`（gather）は `T[]`（可変長）として表現する。キャンバス上では 1 本の線で表現する。

```
入力ブロック（yamaha-els03）              計算ノード
┌────────────────────┐                  ┌───────────────────┐
│ upper.{note}.pressed  ◎[88] ── ── ──►│ vel_bits_{note}   │
│ upper.{note}.velocity ⬦[88] ── ── ──►│  in  ⬦[N]         │
│                                       │  bit_0 ◎[N] ── ──►│ 出力ブロック
└────────────────────┘                  └───────────────────┘
```

（◎ = 二重輪郭の○、⬦ = 二重輪郭の◇）

### 出力ブロックのポート構造

```
出力ブロック（vrchat-default）
┌──────────────────────────────────┐
│ upper_key.{note}.pressed ◎[88]   │  ← keyboard → bool[88]
│ lower_key.{note}.pressed ◎[88]   │
│ expression.value         ◇       │  ← slider → float（scalar）
└──────────────────────────────────┘
```

## テスター

**メタファー: マルチメーター・オシロスコーププローブ**

電子回路のテスターのように、グラフ上の任意のポートにプローブを当てて信号を観測できる機能。入力から出力まで電気がきれいに流れているかを確認する。

### 動作条件

ブリッジが実行中のとき（またはテスト実行モード）にのみ有効。停止中はプローブ操作は行えない。

### プローブ操作

| 操作 | 内容 |
|---|---|
| ポートをホバー | 現在値のツールチップをポップアップ表示。スカラーは値そのもの、配列型は要素数と先頭数件 |
| ポートをクリック（テスターモード中） | プローブを「固定」する。ポート横にピン表示し、常時値を表示し続ける |
| 固定プローブをクリック | 固定を解除 |

### 信号パスのハイライト

特定のポートを選択したとき、その信号が通る経路（上流ノード→接続線→下流ノード）を全てハイライトする。どこで信号が途切れているかを一目で確認できる。

```
  ◎[88]──── ── ──►  ⬦[N] ═══ ══ ══► ◇   ← ハイライト中の経路
              ↑
         プローブ固定中: 0.76
```

### テスターモードの有効化

変換グラフ編集画面のツールバーに「テスト」ボタンを置く。押下するとデバッグ構成ダイアログを開き、使用するデバイス接続設定を確認・編集してからブリッジを内部的に起動し、テスターモードに入る。再押下または停止ボタンで終了する。

#### デバッグ構成ダイアログ

IDE のデバッグ構成に相当するUI。テスト実行のたびに開き、前回の設定を初期値として表示する。設定は preferences にキャッシュされ、次回起動時に自動補完される。

```
─── テスト構成 ─────────────────────────────────────
入力デバイス: [ ELS-03 Series              ▼ ]
出力デバイス: [ 127.0.0.1:9000（VRChat）   ▼ ]

              [ キャンセル ]  [ テスト開始 ]
```

#### ブリッジ同時起動の制限

デバイス構成プレビュー・変換グラフテスター・プロファイル実行のいずれかがブリッジを起動中の場合、別の起動操作はブロックされる。「別のブリッジが動作中です」とダイアログを表示し、現在の起動元画面へ誘導する。画面の導線上、同時起動が発生するケースは実質的に存在しない。

| 操作 | 内容 |

| `G` | 選択ノードの移動（マウスに追従、クリックで確定・右クリックでキャンセル） |
| ポートからドラッグ | 接続線を引く。別ポートにドロップで接続、空きにドロップでキャンセル |

| `X` / `Delete` | 選択ノードの削除 |

| ボックス選択（左ドラッグ） | 範囲内のノードを選択 |
| 右クリック | コンテキストメニュー |

- 保存 / 名前をつけて保存 / 開く

### X クランプ

左→右のデータフロー（[→ 01-principles.md](01-principles.md)）を操作レベルで強制する。

**ワイヤードラッグ中**、先端の X 座標はドラッグ元出力ポートの X を下限とする（それより左には伸びない）。ベジエ曲線の制御点も右向きを維持し、曲線が左に折れ返らないようにする。

**ノードドラッグ中**、ノードの X は接続してくる上流ノードの X の最大値を下限とする。

```
上流 A (x=200) ─┐
                 ├─► 対象ノード  ← x=200 より左にはドラッグできない
上流 B (x=280) ─┘
```

#### 将来拡張：カスケードドラッグ（未実装）

ノードドラッグ時に下流ノードを追従させるモード。preferences でオン/オフを選択可能にする予定。複数上流が衝突するケースの仕様は実装時に検討する。

## 新規作成フロー

ダッシュボードから単体で新規作成した場合は、最初に使用デバイスを宣言するステップを挟む。

```
「変換グラフを新規作成」押下
  ↓
デバイス宣言ダイアログ
  入力デバイス: [ + 追加 ]
    @yamaha-els03/devices/yamaha-els03.yaml  id: yamaha-els03
  出力デバイス: [ + 追加 ]
    @vrchat-osc/devices/vrchat-osc.yaml      id: vrchat-osc
  [ キャンセル ]  [ 作成 ]
  ↓
ブロックが自動生成された状態でエディターを開く
```

プロファイル詳細画面の「マッピングを編集」から遷移した場合は、対応するプロファイルの `inputs` / `outputs` に基づいてブロックが自動生成された状態で開く。

宣言したデバイスは mapper の `input_devices` / `output_devices` に保存される。後から変換グラフエディター内で追加・変更もできる。
