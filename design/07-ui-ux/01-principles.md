# 設計原則

## データの流れ

**入力から出力へ、左から右に進む。** すべての画面でこの方向を統一する。

```
入力デバイス（左）  →  変換・マッピング（中央）  →  出力デバイス（右）
```

---

## レイアウト設計

### 慣習からではなく、構造から導出する

各画面のレイアウトは以下の問いから導出する：

- この画面でユーザーが行う主要な操作・判断は何か
- 情報の関係性（対称・従属・時系列・空間的）はどの構造で表現されるか
- ユーザーの注意はどこから始まりどこへ移動するか

一般的な Web / ネイティブアプリの慣習（ナビゲーションバー・サイドバー・カード一覧・インスペクターペイン等）は出発点にしない。採用する場合はその画面の意味構造との一致を根拠とする。

Midori のフォーム構造はかなり動的であり、ノードプログラミング・リアルタイム監視・動的バインディングエディターなど、既存の慣習パターンが破綻する場面が多い。先に慣習を考えると設計の整合性が失われる。

### 各パーツに説得力を持たせる

レイアウトやパーツの形状・配置には、それが**なぜそこにあるか**の説明責任がある。「よくある UI だから」は根拠にならない。

## メタファー駆動アプローチ

各サーフェスに物理的メタファーを割り当て、そのメタファーから UX を導出することで設計に説得力を持たせる。

### メタファーの明記を必須とする

**画面・コンポーネントの設計ドキュメントには、そのサーフェスが依拠するメタファーを必ず明記する。** 画面全体のメタファーだけでなく、パーツ単位のメタファーが存在する場合もそれぞれ記述する。モックアップへの指示もメタファーを起点とする。

| サーフェス | 物理的メタファー | UX への示唆 |
|---|---|---|
| 全体のデータフロー | ブレッドボード＋導線 | 信号が左→右へ物理的に流れていく。接続は「張る」もの |
| デバイスブロック（変換グラフ） | IC チップ（DIP パッケージ） | 両側にピン（ポート）が並ぶ。折りたたみはチップを伏せる操作 |
| 変換ノード | ロジックゲート・オペアンプ | 小さな回路素子。入力→処理→出力が一体 |
| 接続線 | ジャンパーワイヤー | 被覆の色 = 信号の型。物理的に「つながっている」状態 |
| バインディングエントリ | パッチパネル／端子台 | 端子と端子を対応づける行為。テーブルへの記入ではない |
| プロファイルプレビュー | オシロスコープ・テスト計器 | 信号を「観測する」。介入するものではない |
| レイアウトタブ | コントロールパネル設計 | 操作卓の部品を盤面に置いていく |

メタファーはモックアップ作成の起点として使う。メタファーが完全に一致しない場合でも、「その操作が物理世界ではどういう行為か」を問い続けることが設計の一貫性を保つ。

## 言葉遣いと用語サポート

開発ツールでありながら、楽器奏者など非エンジニアが使うことを想定する。

### 言葉遣いの原則

- UI に表示するラベル・メッセージは**平易な日本語**を優先する
- エラーメッセージは原因と対処を具体的に書く（「エラーが発生しました」ではなく「○○が見つかりません。△△を確認してください」）
- 技術用語（SysEx・CC・OSC アドレス等）は避けられない場合でも、文脈で意味が推測できるよう表現する

### 固有名詞・専門用語のサポート UI

Midori 固有の概念（デバイス構成・変換グラフ・Signal 指定子等）や MIDI/OSC の専門用語には、以下のサポートを設ける。

| サポート形式 | 使用箇所 | 内容 |
|---|---|---|
| ツールチップ（`?` アイコン） | フィールドラベル横 | ホバーで用語の 1〜2 行説明を表示 |
| インライン注釈 | 複雑な設定項目 | フィールドの下に小さなテキストで補足 |
| AI チャットへの誘導 | わかりにくい画面 | 「AI に聞く」ボタンでチャットパネルを開き、文脈付きで質問できる |

## アクセシビリティ

### 色の二重エンコード

**色だけを識別の根拠にしてはならない。** 色覚特性や高コントラスト設定に対応するため、色には必ず別の識別子を併用する。

| 要素 | 色 | 色以外の識別子 |
|---|---|---|
| デバイスブロック（mapper） | デバイス固有のアクセントカラー | ヘッダーに `device_id` ラベル |
| デバイスカード（プレビュー） | 同上 | カードの上辺ボーダー＋ラベル |
| ポート（bool） | muted blue | ○（円形） |
| ポート（float） | muted amber | ◇（菱形） |
| ポート（int） | muted teal | □（四角） |
| ポート（pulse） | muted purple | △（三角） |
| ポート（bool[N]・固定長） | muted blue | ○ の二重輪郭（実線）＋ N バッジ |
| ポート（float[N]・固定長） | muted amber | ◇ の二重輪郭（実線）＋ N バッジ |
| ポート（int[N]・固定長） | muted teal | □ の二重輪郭（実線）＋ N バッジ |
| ポート（bool[]・可変長） | muted blue | ○ の二重輪郭（破線） |
| ポート（float[]・可変長） | muted amber | ◇ の二重輪郭（破線） |
| ポート（int[]・可変長） | muted teal | □ の二重輪郭（破線） |
| 接続線（スカラー） | 接続ポートの型色 | 実線 |
| 接続線（配列型） | 接続ポートの型色 | 破線 |
| ノードカテゴリ | カテゴリ色（背景の薄い tint） | ノード左端のアイコン＋カテゴリラベル |
| エラー状態 | 赤 | ⚠ アイコン＋テキストメッセージ |
| アクティブ状態 | 緑または青 | 点灯アニメーション |
