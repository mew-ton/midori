# プロファイル詳細画面

## メタファー

| サーフェス | メタファー |
|---|---|
| 画面全体 | **パイプライン制御卓** — 設定したパイプラインの全体像を確認し、起動・停止する場所 |
| プレビュータブ | **テストベンチ** — 入力から出力まで電気が正しく流れているかを観測する台 |
| 入力プレビュー（左） | **オシロスコープの入力チャンネル** — 入力信号の現在状態をリアルタイム表示 |
| 出力モニター（右） | **オシロスコープの出力チャンネル** — 出力信号の現在状態をリアルタイム表示 |
| 実行 / 停止ボタン | **電源スイッチ** — パイプラインに通電する・遮断する |
| イベントログ | **シリアルモニター** — パイプライン内を流れるデータの時系列記録 |
| 設定タブ | **配線図** — パイプラインの接続構成（どのデバイスとグラフをどう繋ぐか）を定義する |
| デバイスエントリカード | **端子台の一行** — 各デバイスとその物理接続先を 1 エントリで管理する |

---

プロファイルの概観確認・実行・設定を行う画面。**プレビュー** と **設定** の 2 タブ構成。

---

## プレビュータブ

```
[ 入力プレビュー（左）]              [ 出力モニター（右）]

 ┌─ yamaha-els03 ──────────┐        ┌─ vrchat-default ────────┐
 │  upper keyboard         │        │  upper_key_60 ●         │
 │  ████ ░ ████░░         │        │  upper_key_62 ○         │
 │  lower keyboard         │        │  expression  ──●──      │
 │  pedal                  │        │  ...                    │
 └─────────────────────────┘        └─────────────────────────┘
 ┌─ heart-rate ────────────┐
 │  ♥ 72 bpm              │
 └─────────────────────────┘

              [ ▶ 実行 ]  [ ■ 停止 ]

 ▶ イベントログ                              （デフォルト折りたたみ）
```

複数 inputs / outputs がある場合、デバイスごとにカードを縦に並べる。各カードのヘッダーに `inputs[].id` を表示する。

### イベントログ（展開時）

```
 ▼ イベントログ                    [ クリア ]  [ 停止 | ● 記録中 ]

 ┌─────────────────────────────────────────────────────────────┐
 │  12:34:56.123  raw-event    [yamaha-els03] noteOn ch1 note=60│
 │  12:34:56.124  device-state [yamaha-els03] upper.60.pressed=1│
 │  12:34:56.125  signal       upper.60.pressed = true          │
 │  ...                                                        │
 └─────────────────────────────────────────────────────────────┘
```

| 要素 | 内容 |
|---|---|
| 入力プレビュー（左） | 入力の ComponentState を表示。停止中は静的表示（灰色）。実行中はアクティブなコンポーネントが点灯する |
| 出力モニター（右） | 出力の ComponentState を表示。同上 |
| 実行 / 停止ボタン | ブリッジプロセスを起動・終了する |
| エラー表示 | デバイス未接続は灰色。ランタイムエラーは赤（経路単位で伝播） |
| マッピングを編集 | 変換グラフ編集画面へ遷移 |
| イベントログ | デフォルト折りたたみ。ブリッジ起動中のみ記録される |

### ブリッジの状態表現

| 状態 | 実行ボタン | プレビュー・モニター |
|---|---|---|
| 停止中 | `[ ▶ 実行 ]` 有効 | 灰色で静的表示 |
| 起動中（中間） | `[ ◌ 起動中… ]` 非活性（スピナー） | 灰色のまま |
| 実行中 | `[ ■ 停止 ]` 有効 | 値変化に応じてコンポーネントが点灯 |
| 停止中（中間） | `[ ◌ 停止中… ]` 非活性 | 点灯が止まり灰色に戻る |
| 異常終了 | `[ ▶ 実行 ⚠ ]` 有効 | 灰色で静的表示。イベントログが自動展開 |

### イベントログの詳細

| 要素 | 内容 |
|---|---|
| タイムスタンプ | `HH:mm:ss.SSS` 形式 |
| イベント種別 | `raw-event` / `device-state` / `signal` / `log` / `error-path` の 5 種。種別ごとに色分けする |
| ログ本文 | `raw-event` / `device-state` は `[device_id]` プレフィックスを付ける |
| クリアボタン | 表示中のログをすべて消去する |
| 記録停止ボタン | ログの追記を一時停止する。再クリックで再開 |
| 最大保持件数 | 直近 1000 件を保持。超過分は古い順に破棄する |
| 折りたたみヘッダー | エラー発生時は折りたたみ状態でもヘッダーを赤くして通知する |

## 設定タブ

プロファイルのメタ情報とパイプライン構成を管理する。

```
プロファイル名: [ ___________________________ ]

─── 入力 ─────────────────────────────── [ + 追加 ]
┌──────────────────────────────────────────────────┐
│ id: yamaha-els03  devices/yamaha-els03.yaml  [ × ]│
│   driver: midi                                   │
│   接続: [ ELS-03 Series               ▼ ]        │
└──────────────────────────────────────────────────┘

─── 変換グラフ ────────────────────────────────────
変換グラフ: [ my-avatar.yaml                     ▼ ]

─── 出力 ─────────────────────────────── [ + 追加 ]
┌──────────────────────────────────────────────────┐
│ id: vrchat-default  devices/vrchat-default.yaml  │
│   driver: osc                                   │
│   host: [ 127.0.0.1 ]   port: [ 9000 ]          │
│   アバターパラメーター JSON: [ パスを選択... ]   │
└──────────────────────────────────────────────────┘
```

各エントリはカード形式。`id` は省略時にデバイスファイルのベース名から自動生成され、カード上で編集できる。

### デバイス構成ファイルの選択

`[ + 追加 ]` 押下時、ファイル選択ドロップダウンを表示する。

```
デバイス構成を選択:
  ▼ マイデバイス
      devices/my-custom.yaml
  ▼ yamaha-els03 プラグイン
      @yamaha-els03/devices/yamaha-els03.yaml
  ─────────────────────────────
  [ ファイルを開く... ]
```

「ファイルを開く」ではワークスペース外のファイルも選択できるが、ワークスペース外パスは `preferences.yaml` のアクセス制限外となるため警告を表示する。

### プラグイン由来デバイス構成の扱い

プラグインから提供されたデバイス構成（`@<plugin-name>/...`）をデバイス構成編集画面で開いた場合は **read-only** モードで表示する。「コピーして編集」でワークスペースの `devices/` にコピーしてから編集する。

### driver ごとの接続設定

| driver | 接続設定 |
|---|---|
| `midi` | OS が認識している MIDI デバイスの一覧から選択 |
| `osc` | 出力: host・port を入力 / 入力: listen_port を入力 / 双方向: 全て入力 |

デバイス種別定義 が `additional_fields` を宣言している場合、それらのフィールドが追加で表示される。詳細 → [`10-driver-plugin.md`](../10-driver-plugin.md)
