# エラー・空状態

## 初回起動

初回起動時（`recent.workspaces` が空の場合）、ダッシュボード表示前にワークスペース選択ダイアログを表示する。

```
┌─────────────────────────────────────────┐
│  Midori へようこそ                        │
│                                         │
│  ワークスペース（作業リポジトリ）を        │
│  選択または作成してください               │
│                                         │
│  [ フォルダーを開く... ]                 │
│  [ 新規フォルダーを作成... ]              │
│                                         │
└─────────────────────────────────────────┘
```

選択後、そのフォルダーをワークスペースとして開きダッシュボードへ遷移する。次回起動時は `recent.workspaces` の先頭を自動的に開く。

---

## プロファイルファイルの破損・バリデーションエラー

ダッシュボードからプロファイルを開こうとした際に読み込みエラーが発生した場合、エラーの内容を示すダイアログを表示してダッシュボードに戻る。

---

## デバイスが見つからない

プロファイルが要求するデバイスが OS 上に存在しない場合、プロファイル詳細画面の対応する入力プレビュー / 出力モニターを **灰色** で表示し、実行ボタンを無効化する。

```
 ┌──────────────────┐
 │  upper keyboard  │  ← 灰色（デバイス未接続）
 │  ░░░░ ░ ░░░░░░  │
 │  MIDI デバイスが  │
 │  見つかりません   │
 └──────────────────┘

 [ ▶ 実行 ] ← 無効（グレーアウト）
```

デバイスが接続されると自動的に灰色が解除され、実行ボタンが有効になる。

---

## ブリッジの異常終了

ブリッジプロセスが予期せず終了した場合：

- 実行状態を「異常終了」に遷移し、実行ボタンを `[ ▶ 実行 ⚠ ]` に戻す
- イベントログを自動展開し、最後のログ行とプロセス終了コードを表示する
- 自動再起動は行わない

```
 [ ▶ 実行 ⚠ ]

 ▼ イベントログ                    [ クリア ]
 ┌────────────────────────────────────────────────┐
 │  12:34:56.999  log  error  bridge exited with  │
 │                     code 1                     │
 └────────────────────────────────────────────────┘
```

---

## ランタイムエラー（実行中）

ブリッジ実行中にエラーが発生した場合、経路単位でエラーを伝播させ、該当コンポーネントを **赤** で表示する。イベントログのヘッダーも赤くなり、折りたたんでいても気づける。

---

## 用語の整理

| 用語 | 定義 |
|---|---|
| **デバイス構成** | raw events ↔ ComponentState の変換定義。1 デバイス = 1 ファイル（`devices/*.yaml`） |
| **変換グラフ** | ComponentState → Signal の変換ノードグラフ定義（`mappers/*.yaml`） |
| **プロファイル** | 入力デバイス構成・変換グラフ・出力デバイス構成・実デバイス接続設定を束ねた実行単位 |
| **ドライバー** | 物理 I/O 層を担う外部プロセスプラグイン（MIDI / OSC 等）。すべてプラグインとして動作する |
| **デバイス種別定義** | ドライバーを基底として接続設定・binding の差分を宣言するプラグイン。コードを持たない（例: `osc-vrchat`） |
