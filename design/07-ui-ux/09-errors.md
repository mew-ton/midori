# エラー・空状態

---

## ワークスペース選択画面

### ワークスペースフォルダが見つからない

`recent.workspaces` に記録されたフォルダが削除・移動されていた場合、そのカードをグレーアウトして「フォルダが見つかりません」を表示する。× ボタンで一覧から除去できる。開こうとした場合はエラーダイアログを表示して除去を促す。

## ダッシュボード

空状態（プロファイル 0 件・要素 0 件）は [`04-dashboard.md`](04-dashboard.md) を参照。

## アダプター編集画面

### バインディングが 0 件（新規作成直後）

バインディングタブのリストが空の状態。案内テキストを表示する。

```text
バインディングがありません。
「+ バインディングを追加」でデバイスの入出力を定義してください。
```

### レイアウトが 0 件（新規作成直後）

レイアウトタブのキャンバスが空の状態。案内テキストを表示する。

```text
レイアウトにコンポーネントがありません。
バインディングタブでコンポーネントを定義すると、ここに配置できます。
```

## 変換グラフ編集画面

### ノードが 0 件・接続が 0 本（新規作成直後）

入力・出力ブロックのみ存在し、計算ノードと接続線がない状態。特別な案内は表示しない（ツールボックスが visible であることで操作の起点が自明）。

ただし保存後にプロファイルから実行しようとすると、接続が存在しないことによるバリデーションエラーがプロファイル詳細画面で発生する（[→ プロファイル詳細：参照ファイルのバリデーションエラー](#参照ファイルのバリデーションエラー)）。

## プロファイル詳細画面

### 新規作成直後（デバイス・マッパーが未設定）

プロファイルトップ画面で構成が空の状態。実行ボタンは無効。

```text
─── 入力 ─────────────────────────────── [ + 追加 ]
  （未設定）

─── 変換グラフ ────────────────────────────────────
  変換グラフ: （未設定）

─── 出力 ─────────────────────────────── [ + 追加 ]
  （未設定）
```

入力・出力・変換グラフがすべて設定されるまで実行ボタンはグレーアウトのままとなる。

### 参照ファイルのバリデーションエラー

プロファイルが参照するアダプター YAML またはマッパー YAML が見つからない場合、該当エントリをエラー表示して実行ボタンを無効化する。

```text
─── 入力 ──────────────────────────────────────────
┌──────────────────────────────────────────────────┐
│ ⚠ adapters/yamaha-els03.yaml                      │
│   ファイルが見つかりません                        │
│                              [ 再選択 ]  [ × ]   │
└──────────────────────────────────────────────────┘
```

「再選択」で別のファイルを指定できる。

### デバイスが見つからない（OS 上に存在しない）

プロファイルが要求するデバイスが OS 上に存在しない場合、対応する入力プレビュー / 出力モニターを **灰色** で表示し、実行ボタンを無効化する。

```text
 ┌──────────────────┐
 │  upper keyboard  │  ← 灰色（デバイス未接続）
 │  ░░░░ ░ ░░░░░░  │
 │  MIDI デバイスが  │
 │  見つかりません   │
 └──────────────────┘

 [ ▶ 実行 ] ← 無効（グレーアウト）
```

デバイスが接続されると自動的に灰色が解除され、実行ボタンが有効になる。

### プロファイルファイルの破損・バリデーションエラー

ダッシュボードからプロファイルを開こうとした際に読み込みエラーが発生した場合、エラーの内容を示すダイアログを表示してダッシュボードに戻る。

### ブリッジの異常終了

ブリッジプロセスが予期せず終了した場合：

- 実行状態を「異常終了」に遷移し、実行ボタンを `[ ▶ 実行 ⚠ ]` に戻す
- イベントログを自動展開し、最後のログ行とプロセス終了コードを表示する
- 自動再起動は行わない

```text
 [ ▶ 実行 ⚠ ]

 ▼ イベントログ                    [ クリア ]
 ┌────────────────────────────────────────────────┐
 │  12:34:56.999  log  error  bridge exited with  │
 │                     code 1                     │
 └────────────────────────────────────────────────┘
```

### ランタイムエラー（実行中）

ブリッジ実行中にエラーが発生した場合、経路単位でエラーを伝播させ、該当コンポーネントを **赤** で表示する。イベントログのヘッダーも赤くなり、折りたたんでいても気づける。

## 用語の整理

| 用語 | 定義 |
|---|---|
| **アダプター** | raw events ↔ ComponentState の変換定義。1 デバイス = 1 ファイル（`adapters/*.yaml`） |
| **変換グラフ** | ComponentState → Signal の変換ノードグラフ定義（`mappers/*.yaml`） |
| **プロファイル** | 入力アダプター・変換グラフ・出力アダプター・実デバイス接続設定を束ねた実行単位 |
| **ドライバー** | 物理 I/O 層を担う外部プロセスプラグイン（MIDI / OSC 等）。すべてプラグインとして動作する |
| **アダプター種別定義** | ドライバーを基底として接続設定・binding の差分を宣言するプラグイン。コードを持たない（例: `osc-vrchat`） |
