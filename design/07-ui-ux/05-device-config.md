# デバイス構成編集画面

## メタファー

| サーフェス | メタファー |
|---|---|
| 画面全体 | **配線作業台** — デバイスの端子（raw events）と内部モデル（ComponentState）を接続する作業場 |
| バインディングタブ | **パッチパネル／端子台** — 端子と端子を結線する。テーブルへの記入ではなく「つなぐ」行為 |
| SysEx パターンエディター | **バイトシーケンサー** — バイト列を並べてパターンを組み上げる |
| レイアウトタブ | **コントロールパネル設計台** — 操作卓の部品を盤面に配置する |
| ウィジェット（プレビュー） | **オシロスコープのプローブ表示** — 各コンポーネントの現在値をリアルタイムに観測する正方形の計器 |

---

デバイス構成を定義する画面。タブで **バインディング** / **レイアウト** / **プレビュー** の 3 つのモードを切り替える。

- 保存 / 名前をつけて保存 / 開く

> **方針**: YAML を直接編集することは基本的にない。このエディター画面がデバイス構成の主要な編集 UI である。

## 新規作成時のデバイスタイプ選択

デバイス構成の新規作成時にダイアログで**作成するデバイスの種類**を選択する。一度選択したら変更不可。

選択肢は「どのドライバーがインストールされているか」ではなく、**「ユーザーが設定対象にできるもの」** を提示する。デバイス種別定義（`osc-vrchat` 等）も一級の選択肢として表示する。

```
┌──────────────────────────────────────┐
│ デバイスの種類を選択                  │
│                                      │
│  ○ MIDI 機器                          │
│    MIDI キーボード・音源・コントローラー  │
│                                      │
│  ○ OSC 機器                           │
│    汎用 OSC 送受信                     │
│                                      │
│  ○ VRChat アバター（OSC）              │
│    VRChat のアバターパラメーター制御    │
│                                      │
│  ── インストール済みプラグイン ──      │
│  ○ ...                               │
│                                      │
│              [ キャンセル ][ 作成 ]   │
└──────────────────────────────────────┘
```

- 各選択肢にはデバイス種別の説明を添える
- プラグインが提供する デバイス種別定義 はセクションを分けて表示する
- 選択内容に応じて内部的に `driver` と `device_kind` が決定される（ユーザーはこれらの技術的詳細を意識しない）

## デバイス構成の設定パネル

編集画面内に歯車アイコン等で開けるプレファレンス的なパネルを用意する。`driver` 以外のメタ情報をここで管理する。

| 設定項目 | 内容 |
|---|---|
| 名前 | デバイス構成の表示名 |
| 説明 | 任意の説明文 |
| direction | `any`（デフォルト）/ `input` / `output`。ハードウェアの制約として方向が固定されるデバイスのみ変更する |
| デバイス種別 | 作成時に選択した種別を表示のみ（変更不可）。「MIDI 機器」「VRChat アバター（OSC）」等の表示名で示す。内部的な `driver`/`device_kind` は表示しない |

## バインディングタブ

raw events と ComponentState の対応（binding）、およびコンポーネント構成（definition）を定義する。

> **設計メモ**: 以下のレイアウトはテキストベースの設計段階で自然に収束したテーブル構造であり、最終解ではない。「バインディングを定義する」という行為のメタファーは**パッチパネル／端子台**（端子と端子を対応づける）が適切と考えられる。モックアップ段階でメタファーに基づく再設計を推奨する。
>
> 候補案:
> - **パッチパネル案**: 左列に raw event の端子、右列に definition パスの端子。端子間をドラッグして線を引き接続する。変換ルール（setMap / expr）は線の途中に素子として挿入する
> - **端子台案**: 各行が「一本の結線」として完結。from / to は端子のラベル、モードセレクターは変換素子の種類。行追加 = 端子台にネジ止め

### レイアウト（現行案）

入力セクションと出力セクションを縦に並べる。各セクションは **バインディングエントリのリスト** で構成される。1 エントリ = `from`（左）+ `to`（右）の対。

```
┌─────────────────────────────────────────────────────────────┐
│ [ 入力 ]                                          [ + 追加 ] │
│                                                             │
│  ┌──────────────────────────┬──────────────────────────┐   │
│  │ from（raw event）         │ to（definition）          │   │
│  ├──────────────────────────┼──────────────────────────┤   │
│  │ noteOn  ch1              │ upper.{note}.pressed      │   │
│  │                          │ set: true                 │   │
│  ├──────────────────────────┼──────────────────────────┤   │
│  │ sysex                    │ expression.value          │   │
│  │ [F0][43][70][70][40][5A] │ setMap source: arg1       │   │
│  │ [{arg1}][F7]             │ linear [0x00,0x7F]→[0,1] │   │
│  └──────────────────────────┴──────────────────────────┘   │
│                                                             │
├─────────────────────── divider ─────────────────────────────┤
│ [ 出力 ]                                          [ + 追加 ] │
│  ...（出力は左右が反転: definition 左 / raw events 右）      │
└─────────────────────────────────────────────────────────────┘
```

### 入力セクション

| 列 | 内容 |
|---|---|
| 左列（`from`） | raw event の種別・条件。ドライバー種別（MIDI / OSC 等）によって入力項目が変わる |
| 右列（`to`） | definition のコンポーネント value へのパスと、`set` / `setMap` / `set.expr` の設定 |

### 出力セクション

入力セクションの左右を反転した構造（`from` が definition 側、`to` が raw events 側）。

出力側の `from` 列には、エントリ単位で **プレビュートリガー** を置ける。押下すると対応する raw event をテスト送信し、出力デバイスの実際の応答を確認できる。

### raw events の追加・削除・編集

| 操作 | 内容 |
|---|---|
| `+` ボタン | raw event を手動入力で追加する |
| 各エントリの編集 | インラインで値を直接編集できる |
| 各エントリの削除 | エントリ右の削除ボタン、または選択して `Delete` |

binding が張られているエントリを削除した場合は警告を表示する（binding ごと削除するか確認する）。

### SysEx パターンエディター（左列 / `from`）

`type: sysex` の raw event を選択または追加したとき、左列にパターンエディターを表示する。

```
  [ 0xF0 ][ 0x43 ][ 0x70 ][ 0x70 ][ 0x40 ][ 0x5A ][ {arg1} ][ 0xF7 ]  [ + ]
```

| 要素 | 内容 |
|---|---|
| バイトセル | 16 進（`0xF0`）または 10 進（`240`）で入力。整数値として保存される |
| `{name}` セル | 任意の 1 バイトにマッチするキャプチャ変数。`{arg1}` / `{lo}` 等を入力する |
| セルの追加・削除・並べ替え | `+` で末尾追加。セル右クリックまたはドラッグで操作 |
| キャプチャ変数への切り替え | セル上の `⋮` メニューから「キャプチャ変数に変更」で `{name}` に切り替え可 |

### `set` / `setMap` / `set.expr` モードの切り替え

各エントリの右列（`to`）に、セット方法を切り替えるモードセレクターを置く。

| モード | 使用条件 | 右列の表示 |
|---|---|---|
| `set`（直値） | raw event の値型とコンポーネント value の型が一致する場合 | 固定値の入力欄 |
| `setMap`（条件マップ） | 型が不一致、またはキャプチャ変数から条件分岐が必要な場合 | source + 条件リスト |
| `set.expr`（式） | SysEx の複数キャプチャ変数を計算して 1 値にする場合 | 式入力欄 |

SysEx エントリはキャプチャ変数が存在するため、`set` への直接代入は不可。`setMap` または `set.expr` のみ選択できる。

### setMap エディター（右列 / `to`）

```
  右列
  ┌─────────────────────────────────────┐
  │ to: upper_sustain.pressed           │
  │ mode: [ setMap        ▼ ]           │
  │                                     │
  │ source: [ value       ▼ ]           │
  │                                     │
  │  when: 0 ~ 63    →  set: false  [ × ]  │
  │  when: >= 64     →  set: true   [ × ]  │
  │                                     │
  │  [ + 条件を追加 ]                    │
  └─────────────────────────────────────┘
```

| 要素 | 内容 |
|---|---|
| `source` | どのキャプチャ変数を条件の入力値として使うか。SysEx でキャプチャ変数が存在する場合は必須。CC / pitchBend 等の単一値イベントでは非表示 |
| `when` | 条件式。`64`（完全一致）/ `< 64` / `>= 64` / `0~63`（範囲）で入力する |
| `set` | 条件に一致したときにコンポーネント value にセットする値 |
| 行の順序 | 上から順に評価。ドラッグで並べ替えられる |
| `+ 条件を追加` | 新しい条件行を追加する |

### set.expr エディター（右列 / `to`）

```
  右列
  ┌───────────────────────────────────────┐
  │ to: tempo.value                       │
  │ mode: [ set.expr      ▼ ]             │
  │                                       │
  │  expr: (hi << 2) | ((lo >> 5) & 0x03) │
  │                                       │
  │  使用可能な変数: lo, hi               │
  └───────────────────────────────────────┘
```

| 要素 | 内容 |
|---|---|
| `expr` 入力欄 | 式を直接入力。`set.expr` の構文に準拠 |
| 使用可能な変数 | `from.pattern` で定義されたキャプチャ変数名を一覧表示（参照用） |
| 構文チェック | ロード前にエラーをインラインで表示する |

### raw events の追加 UI とサジェスト

```
  [ noteOn ch1 note=_______________ ]
  ┌─────────────────────────────────┐
  │  noteOn ch1 note=60 vel=80      │
  │  noteOn ch1 note=62 vel=64      │
  └─────────────────────────────────┘
```

サジェストがない場合はドロップダウンを表示しない。

### デバイスタイプごとのサジェスト設計

| デバイスタイプ | direction | サジェストの源泉 |
|---|---|---|
| `midi` | input | 過去の手動入力値 / 編集中に受信した MIDI 信号 |
| `midi` | output | 過去の手動入力値のみ |
| `osc` | input | 過去の手動入力値 / 編集中に受信した OSC パケット |
| `osc` | output | 過去の手動入力値のみ |
| `osc`（device_kind: `osc-vrchat`） | input | VRChat アバターパラメーター JSON / 編集中に受信した OSC パケット |
| `osc`（device_kind: `osc-vrchat`） | output | VRChat アバターパラメーター JSON |

### `osc-vrchat` デバイス種別定義

VRChat はアップロード済みアバターの OSC パラメーター一覧を JSON ファイルとして自動生成する。これを読み込むことで出力側の raw events サジェストを提供できる。

| 要素 | 内容 |
|---|---|
| JSON の読み込み | プロファイル詳細設定画面でファイルパスを指定する。または VRChat のデフォルト出力先を自動検索する |
| サジェストの内容 | JSON に含まれるパラメーター名・型をサジェストとして一覧表示する |
| input 側への流用 | 同じ JSON を入力側のサジェストにも使える場合は共用する |

詳細は [`config/drivers/osc-vrchat.md`](../../config/drivers/osc-vrchat.md) および [`10-driver-plugin.md`](../10-driver-plugin.md) を参照。

## レイアウトタブ

Preview / Monitor の描画構成（`layout` セクション）を定義する。

左ペインに definition パネルを置き、コンポーネントをキャンバスにドラッグ＆ドロップして並べる。

> **ウィジェットの外観はドライバーが提供する。** 本タブが定義するのは「どのコンポーネントをどのウィジェットで描画し、どう配置するか」であり、ウィジェット内部の描画はドライバープラグインの責任範囲である。

ウィジェット仕様の詳細 → [`10-widgets.md`](10-widgets.md)

### グループモデル

レイアウトの配置は **グループの並び順** と **グループ内のアイテムの並び順** だけで決まる。座標は持たない。

```
グループ（named container）
  └── アイテム（widget）が左から右に並び、溢れると折り返す
```

グループ間ギャップ = `2u`、アイテム間ギャップ = `1u`。すべてのプリミティブは `2u × 2u` の正方形、キーボードは `N_semitones × 2u`（幅可変・高さ固定）。詳細 → [`10-widgets.md`](10-widgets.md)

---

### ウィジェットの 3 カテゴリ

コンポーネントの種類によって、ウィジェットの選択権とサイズルールが異なる。

#### カテゴリ A：プラグイン定義のコンポーネント型

`keyboard` のようなプラグインが独自に定義したコンポーネント型。ウィジェットの選択肢はプラグインが固定する（ユーザーが別のウィジェットを選ぶことはできない）。サイズ・内部レイアウトもプラグインの定義に従う。

| 例 | プラグイン | ウィジェット（固定） |
|---|---|---|
| `keyboard` | MIDI ドライバー | キーグリッド（N × M）。N = `key_range` 長、M = children 数 |

#### カテゴリ B：プリミティブ値

スカラーの基本型（`bool` / `pulse` / `int` / `float`）。ユーザーは型に適合するウィジェットを選べるが、**サイズは常に `2u × 2u` 固定。スケーリング・サイズ変更は不可。**

| 型 | 選択可能なウィジェット | 表示 |
|---|---|---|
| `bool` / `pulse` | filled-square | false: 空 / true: `--color-active` で塗りつぶし |
| `int` / `float` | bar（縦棒） | 値を高さで表現（下端 = min、上端 = max） |
| `int` / `float` | dot（XY プロット） | 正方形内に 1 点をプロット（2 軸に対して 1 値の場合は縦軸のみ） |

#### カテゴリ C：固定長配列型

`T[N]`（固定長配列）。入出力コンテキストでは常に固定長であるため、N は定義から逆算できる。

**レンダリング方式: 自動グリッドグループ**

各要素を対応するプリミティブウィジェット（1×1 固定）に展開し、それを内包する**グリッドコンテナ**として配置する。全要素が同サイズのセルであるため、グリッドが自然に適合する。コンテナのサイズは要素数 × セルサイズから自動算出される。サイズ・スケーリングの明示的な指定は不要。

```
bool[5]  → ┌──┬──┬──┬──┬──┐  （columns: 5）
           └──┴──┴──┴──┴──┘

bool[6]  → ┌──┬──┬──┐        （columns: 3, rows: 2）
           ├──┼──┼──┤
           └──┴──┴──┘
```

ユーザーが設定できるのはコンテナのグリッドプロパティのみ：

| 設定 | 内容 |
|---|---|
| `columns` | 1 行あたりの列数。残りは自動で行に折り返す |
| `gap` | セル間の余白（rem 単位） |

### レイアウトキャンバス

```
[ definition（左）]              [ レイアウトキャンバス（右）]

  ┌───────────────────────┐      ┌────────────────────────────────────┐
  │ + コンポーネントを追加  │      │                                    │
  │                       │      │  [■■■■■■■■■]  upper.pressed        │
  │ ▼ upper  keyboard ⋮   │      │  (keyboard, プラグイン固定)          │
  │     velocity  float   │drag► │                                    │
  │     pressed  bool[88] │      │  [□][□][□]  upper.velocity         │
  │ ► expression  float ⋮ │      │  (bool[88] → grid, columns=∞)     │
  └───────────────────────┘      │                                    │
                                 │  [■]  expression                   │
                                 │  (float → bar, 1×1)                │
                                 └────────────────────────────────────┘

                                  選択中: upper.velocity（bool[88]）
                                  ┌────────────────────────────────────┐
                                  │ カテゴリ: 固定長配列（bool[88]）    │
                                  │ columns: [ 16 ]                    │
                                  │ gap:    [ 0.125rem ]               │
                                  └────────────────────────────────────┘
```

| 要素 | 内容 |
|---|---|
| definition パネル（左） | コンポーネントの一覧。キャンバスへドラッグして layout に追加する |
| レイアウトキャンバス（右上） | layout をビジュアルレンダリング。ドラッグで順番・グループ変更 |
| プロパティパネル（右下） | カテゴリに応じた設定項目を表示する（後述） |

### プロパティパネルの表示内容（カテゴリ別）

| カテゴリ | 表示内容 |
|---|---|
| A: プラグイン定義 | カテゴリ名のみ表示。設定項目なし（プラグイン固定） |
| B: プリミティブ | ウィジェット種別セレクター（型に適合するもののみ）。サイズ設定なし |
| C: 固定長配列 | `columns` / `gap` のグリッド設定のみ |

### definition パネルの操作

| 操作 | 内容 |
|---|---|
| キャンバスへドラッグ | layout への追加。1 コンポーネント = 1 エントリ（複数配置不可） |
| `⋮` メニュー | コンポーネントの編集・削除 |
| `+ コンポーネントを追加` | 新しいコンポーネントを definition に追加する |

definition から削除したコンポーネントが layout に残っている場合は警告表示する。

### キャンバスの操作

| 操作 | 内容 |
|---|---|
| ドラッグ＆ドロップ（definition → キャンバス） | layout に追加 |
| ドラッグ＆ドロップ（キャンバス内） | 順番・親子関係の変更 |
| グループに重ねてドロップ | そのグループの子として追加 |
| グループ追加 | 右クリックメニュー「グループを追加」 |

## プレビュータブ（デバッグ用）

バインディングの動作確認をプロファイル不要で行えるデバッグ機能。

### テスト接続設定

プロファイルとは独立した一時的な接続設定。デバイス構成 YAML には保存されない（preferences にキャッシュする）。

```
─── テスト接続 ─────────────────────────────────────
driver: midi（変更不可。バインディングタブで設定済み）
接続: [ ELS-03 Series                          ▼ ]

[ ▶ プレビュー開始 ]  [ ■ 停止 ]
```

「プレビュー開始」は、このデバイスをパススルーマッパーに接続した最小限のブリッジを内部的に起動する。接続設定は preferences に保存され、次回以降自動補完される。

### ライブプレビュー

ブリッジ起動中は layout に基づいてリアルタイムに値が更新される。

```
（停止中: 灰色）           （実行中: アクティブなコンポーネントが点灯）

 ████ ░ ████░░  upper      ████ ● ████░●  upper
 ████ ░ ████░░  lower      ████ ░ ████░░  lower
 ██░░           pedal      ██░░           pedal
 ──●  expression           ────●  expression
```

`direction: output` のデバイス構成の場合は Monitor 表示となり、バインディングタブの「プレビュートリガー」と組み合わせて出力の確認ができる。
