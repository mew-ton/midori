## セキュリティ水準

Midori 全体で「どの程度まで守るか」の共通基準。各領域のドキュメントはこの水準を参照して目標を設定する。

| 水準 | 内容 |
|---|---|
| **L0** | ユーザーへの警告のみ。技術的な強制なし |
| **L1** | 問題を **検知** できる（ログ・監視） |
| **L2** | 偶発的な侵害を **防止** できる。悪意ある攻撃には限定的 |
| **L3** | 悪意ある攻撃者が宣言された権限以上には動けない |

L3 はクロスプラットフォームで均一に達成するのが難しい。L1 → L2 を現実的な目標に置く。

## 共通の設計方針

- **最小権限原則**：コンポーネントには必要な権限のみを与える
- **明示的な権限宣言**：プラグイン・AI 機能等が必要とするアクセスは事前に宣言させる
- **段階的採用**：L1 → L2 → L3 と段階的に進める
- **クロスプラットフォームの均一性**：可能な範囲で 3 OS（macOS / Linux / Windows）で同等の保護を提供する

## プラグイン信頼モデル

「コードを持たない = 安全」という単純な二分では不十分。YAML 設定のみのプラグインも構成パターンによってはセキュリティ上の問題を引き起こしうる。
