# ダッシュボード

## メタファー

| サーフェス | メタファー |
|---|---|
| 画面全体 | **機材ラック** — ステージに持ち込む機材を並べたラック。上段に本番用ユニット（プロファイル）、中段に部品モジュール（デバイス等）、下段に設定・管理系 |
| プロファイルカード | **ラックユニット** — 電源を入れれば動き出す、完結したパイプライン設定 |
| 要素リスト | **部品棚** — ラックユニットを組み立てるための部品を並べた棚 |
| インフラ帯 | **ラック背面パネル** — 電源・接続設定など、通常は触らないが必要なときに手を伸ばす場所 |

---

同一のレイアウト骨格（大セクション ／ 小セクション ／ インフラ帯）を、**ワークスペース選択画面**と**ダッシュボード**の 2 つの状態で共用する。インフラ帯は両状態で共通。

| 状態 | 大セクション | 小セクション |
|---|---|---|
| ワークスペース選択画面 | ワークスペース一覧 | なし（省略） |
| ダッシュボード | プロファイル一覧 | デバイス・ドライバー・デバイス種別一覧 |

---

## ワークスペース選択画面

アプリ起動時に表示するアプリレベルの画面。ワークスペースを選んだ後にダッシュボードへ遷移する。

```
┌──────────────────────────────────────────────────────────┐
│  ワークスペース       [ フォルダを開く ] [ + 新規作成 ]   │  ▲
│                                                          │  │
│  ┌───────────────────┐  ┌───────────────────┐           │  │ 大
│  │ my-els03-setup    │  │ vrchat-tools      │           │  │
│  │ ~/Documents/…     │  │ ~/Projects/…      │           │  │
│  │          [開く][×] │  │          [開く][×] │           │  ▼
│  └───────────────────┘  └───────────────────┘           │
├──────────────────────────────────────────────────────────┤
│  [ 🔌 プラグイン管理 ]   [ ⚙ Preferences ]               │  最小
└──────────────────────────────────────────────────────────┘
```

### ワークスペースカード

| 要素 | 内容 |
|---|---|
| 名前 | `.midori/workspace.yaml` の `name` フィールド（未設定時はフォルダ名） |
| パス | ワークスペースのフォルダパス |
| 開くボタン | ダッシュボードへ遷移 |
| 削除ボタン（×） | 最近の一覧から除去（フォルダ自体は削除しない） |

最後に開いた順でソート。カードは 2 列グリッド。

### フォルダを開く

OS のフォルダ選択ダイアログを表示する。選択したフォルダに `.midori/workspace.yaml` が存在しない場合は初期化ダイアログを表示する：

```
このフォルダをワークスペースとして初期化しますか？
.midori/workspace.yaml を作成します。

[ キャンセル ]  [ 初期化して開く ]
```

### 新規作成

OS のフォルダ選択ダイアログで保存先を選び、`.midori/workspace.yaml` を生成してダッシュボードへ遷移する。

### 空状態

ワークスペースが 0 件のとき：

```
ワークスペースがありません。
フォルダを開くか、新規作成してください。

[ フォルダを開く ]  [ + 新規ワークスペースを作成 ]
```

## ダッシュボード

アプリ起動・ワークスペースを開いた後のホーム画面。**プロファイル（大）／ 要素（小）／ インフラ（最小）** の 3 段構成。

ランタイムとして「実行する」導線をメインに、IDE として「作る・編集する」導線をサブに配置する。

## プロファイルセクション（大）

ワークスペース内のプロファイルを**最後に開いた順**でカード表示する。

### プロファイルカード

| 要素 | 内容 |
|---|---|
| 名前 | `profiles/*.yaml` の `name` フィールド（未設定時はファイル名） |
| 説明 | `description` フィールド（未設定時は空欄） |
| 実行ボタン | プロファイル詳細画面（プレビュータブ）へ遷移し、ブリッジを起動 |
| 編集ボタン | プロファイル詳細画面（設定タブ）へ遷移 |
| 削除ボタン | 確認ダイアログを経てファイルを削除 |

カードは 2 列グリッド。ウィンドウ幅が狭い場合は 1 列に折りたたむ。

ブリッジが実行中のプロファイルは、カードに「● 実行中」バッジを表示し、実行ボタンを「詳細を見る」に切り替える。ブリッジ同時起動の制限（[→ `06-graph.md`](06-graph.md)）により、他のカードの実行ボタンは非活性になる。

### 新規作成

「+ 新規作成」押下で選択ダイアログを表示する。

```
─── プロファイルを作成 ──────────────────────────────
  ● 空から作成
    デバイスと変換グラフを一から設定します

  ○ コピーして作成
    既存プロファイルの設定・変換グラフをそのまま複製します
    [ My Profile           ▼ ]

[ キャンセル ]  [ 作成 ]
```

| 選択肢 | 内容 |
|---|---|
| 空から作成 | デバイス・変換グラフが未設定のプロファイルを生成してトップ画面へ遷移 |
| コピーして作成 | 選択したプロファイルのデバイス設定と変換グラフファイルをそれぞれ複製して新規プロファイルを生成し、トップ画面へ遷移 |

「コピーして作成」はプロファイルが 1 件以上存在する場合のみ選択できる。

### 空状態

プロファイルが 0 件のとき、カード領域の代わりに以下を表示する：

```
まだプロファイルがありません。
デバイスと変換グラフを接続する実行設定を作成しましょう。

[ + プロファイルを作成する ]
```

## 要素セクション（小）

ワークスペース内の device / driver / device_kind をリスト表示する。

mapper はここに出ない。mapper はプロファイルに紐づく要素であり、プロファイル詳細画面の「マッピングを編集」から入る。

### 要素リスト

各行：名前（`name` フィールド、未設定時はファイル名）・タイプバッジ（`device` / `driver` / `device_kind`）・編集・削除。

ソート順：タイプ（device → driver → device_kind）→ 名前の昇順。

### デバイスを作る

「+ デバイスを作る」押下でドライバー選択ダイアログを表示し、デバイス構成編集画面（空の状態）へ遷移。

driver / device_kind の新規作成はここには置かない（プラグイン開発の文脈であり、通常ユーザーの操作導線から外れる）。

### 空状態

要素が 0 件のとき：

```
デバイス構成がありません。
デバイスを作成すると、変換グラフで利用できます。

[ + デバイスを作る ]
```

## インフラ帯（最小）

| ボタン | 遷移先 |
|---|---|
| 🔌 プラグイン管理 | Preferences 設定画面 > プラグインタブ |
| ⚙ Preferences | Preferences 設定画面 > 一般タブ |

ボタン 2 つのみのコンパクトな帯。カード・リストは持たない。
