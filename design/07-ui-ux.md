# UI / UX 要件

## 基本デザインルール

**データの流れは左から右に進む。** すべての画面でこの方向を統一する。

```
入力デバイス（左）  →  変換・マッピング（中央）  →  出力デバイス（右）
```

### カラーシステム

コンポーネントの状態表示に使う色を統一する。

| 色 | 意味 | 使用箇所 |
|---|---|---|
| 灰色 | 入出力の検証ができないが定義はある | 対応デバイスが未接続 / ブリッジ停止中の静的表示 |
| 赤 | エラー | プレビュー：対応デバイスが見つからない。実行中：ノードグラフのエラー・ゼロ除算等のランタイムエラー |
| 緑または青 | アクティブ（その瞬間に値が変化した） | 実行中のみ有効。入出力に影響のあったコンポーネントを瞬間的に点灯させる。停止中・プレビュー時は使用しない |

## 画面構成と遷移

```
ダッシュボード
├── デバイス構成を新規作成 ──→ デバイス構成編集画面
│                                ├── バインディングタブ
│                                └── レイアウトタブ
├── 最近のデバイス構成 ─────→ デバイス構成編集画面
├── プロファイルを新規作成 ─→ プロファイル詳細画面（設定タブ）
└── 最近のプロファイル ────→ プロファイル詳細画面
                                 ├── プレビュータブ
                                 │     └── マッピングを編集 ──→ 変換グラフ編集画面
                                 └── 設定タブ
```

## ダッシュボード

アプリ起動時のホーム画面。

| 要素 | 内容 |
|---|---|
| デバイス構成を新規作成 | ドライバー選択ダイアログを表示し、選択後にデバイス構成編集画面（空の状態）へ遷移 |
| 最近のデバイス構成一覧 | 最近作成・編集したデバイス構成をカード形式で表示。クリックでデバイス構成編集画面へ遷移 |
| プロファイルを新規作成 | プロファイル詳細画面（設定タブ）へ遷移 |
| 最近のプロファイル一覧 | 最近作成・使用したプロファイルをカード形式で表示。クリックでプロファイル詳細画面へ遷移 |

## デバイス構成編集画面

デバイス構成を定義する画面。入力・出力を同一画面内に上下または左右で並べる。

タブで **バインディング** と **レイアウト** の2つの編集モードを切り替える。

- 保存 / 名前をつけて保存 / 開く

#### 新規作成時のドライバー選択

デバイス構成の新規作成時にダイアログでドライバーを選択する。一度選択したら変更不可。

```
┌─────────────────────────────┐
│ ドライバーを選択              │
│                             │
│  ○ midi                     │
│  ○ osc                      │
│  ○ osc-vrchat               │
│                             │
│         [ キャンセル ][ 作成 ]│
└─────────────────────────────┘
```

#### デバイス構成 の設定パネル

編集画面内に歯車アイコン等で開けるプレファレンス的なパネルを用意する。`driver` 以外のメタ情報をここで管理する。

| 設定項目 | 内容 |
|---|---|
| 名前 | デバイス構成 の表示名 |
| 説明 | 任意の説明文 |
| direction | `any`（デフォルト）/ `input` / `output`。ハードウェアの制約として方向が固定されるデバイスのみ変更する |
| driver | 選択済みの値を表示のみ（変更不可） |

### バインディングタブ

raw events と ComponentState の対応（binding）、およびコンポーネント構成（definition）を定義する。

#### レイアウト

入力セクションと出力セクションを縦に並べる。情報量が多いため縦スクロールで対応し、セクション間にはディバイダーを置く。

各セクションは **バインディングエントリのリスト** で構成される。1エントリ = `from`（左）+ `to`（右）の対。左右は 1:1 対応のため、ワイヤーによる接続操作は行わない。

```
┌─────────────────────────────────────────────────────────────┐
│ [ 入力 ]                                          [ + 追加 ] │
│                                                             │
│  ┌──────────────────────────┬──────────────────────────┐   │
│  │ from（raw event）         │ to（definition）          │   │
│  ├──────────────────────────┼──────────────────────────┤   │
│  │ noteOn  ch1              │ upper.{note}.pressed      │   │
│  │                          │ set: 1                    │   │
│  ├──────────────────────────┼──────────────────────────┤   │
│  │ sysex                    │ expression.value          │   │
│  │ [F0][43][70][70][40][5A] │ setMap source: arg1       │   │
│  │ [{arg1}][F7]             │ linear [0x00,0x7F]→[0,1] │   │
│  ├──────────────────────────┼──────────────────────────┤   │
│  │ sysex                    │ tempo.value               │   │
│  │ [F0][43]...[{lo}][{hi}]  │ expr: (hi<<2)|((lo>>5)   │   │
│  │ [F7]                     │       &0x03)              │   │
│  └──────────────────────────┴──────────────────────────┘   │
│                                                             │
├─────────────────────── divider ─────────────────────────────┤
│ [ 出力 ]                                          [ + 追加 ] │
│  ...（出力は左右が反転: definition 左 / raw events 右）      │
└─────────────────────────────────────────────────────────────┘
```

#### 入力セクション

| 列 | 内容 |
|---|---|
| 左列（`from`） | raw event の種別・条件。ドライバー種別（MIDI / OSC 等）によって入力項目が変わる |
| 右列（`to`） | definition のコンポーネント value へのパスと、`set` / `setMap` / `set.expr` の設定 |

#### 出力セクション

入力セクションの左右を反転した構造（`from` が definition 側、`to` が raw events 側）。

出力側の `from` 列には、エントリ単位で **プレビュートリガー** を置ける。押下すると対応する raw event をテスト送信し、出力デバイス（楽器等）の実際の応答を確認できる。

#### raw events の追加・削除・編集

raw events の内容はデバイスタイプ（MIDI / OSC 等）によって異なるが、構造はシンプルに「取りうる値のリスト」として扱う。

| 操作 | 内容 |
|---|---|
| `+` ボタン | raw event を手動入力で追加する |
| 各エントリの編集 | インラインで値を直接編集できる |
| 各エントリの削除 | エントリ右の削除ボタン、または選択して `Delete` |

binding が張られているエントリを削除した場合は警告を表示する（binding ごと削除するか確認する）。

#### SysEx パターンエディター（左列 / `from`）

`type: sysex` の raw event を選択または追加したとき、左列にパターンエディターを表示する。

```
  [ 0xF0 ][ 0x43 ][ 0x70 ][ 0x70 ][ 0x40 ][ 0x5A ][ {arg1} ][ 0xF7 ]  [ + ]
```

| 要素 | 内容 |
|---|---|
| バイトセル | 16進（`0xF0`）または10進（`240`）で入力。整数値として保存される |
| `{name}` セル | 任意の1バイトにマッチするキャプチャ変数。`{arg1}` / `{lo}` 等を入力する |
| セルの追加・削除・並べ替え | `+` で末尾追加。セル右クリックまたはドラッグで操作 |
| キャプチャ変数への切り替え | セル上の `⋮` メニューから「キャプチャ変数に変更」で `{name}` に切り替え可 |

#### `set` / `setMap` / `set.expr` モードの切り替え

各エントリの右列（`to`）に、セット方法を切り替えるモードセレクターを置く。

| モード | 使用条件 | 右列の表示 |
|---|---|---|
| `set`（直値） | raw event の値型とコンポーネント value の型が一致する場合 | 固定値の入力欄 |
| `setMap`（条件マップ） | 型が不一致、またはキャプチャ変数から条件分岐が必要な場合 | source + 条件リスト |
| `set.expr`（式） | SysEx の複数キャプチャ変数を計算して1値にする場合 | 式入力欄 |

SysEx エントリはキャプチャ変数が存在するため、`set` への直接代入は不可。`setMap` または `set.expr` のみ選択できる。

#### setMap エディター（右列 / `to`）

エントリが `setMap` モードのとき、右列にインラインでエディターを表示する。

```
  右列
  ┌─────────────────────────────────────┐
  │ to: upper_sustain.pressed           │
  │ mode: [ setMap        ▼ ]           │
  │                                     │
  │ source: [ value       ▼ ]           │
  │                                     │
  │  when: 0 ~ 63    →  set: 0  [ × ]  │
  │  when: >= 64     →  set: 1  [ × ]  │
  │                                     │
  │  [ + 条件を追加 ]                    │
  └─────────────────────────────────────┘
```

| 要素 | 内容 |
|---|---|
| `source` | どのキャプチャ変数を条件の入力値として使うか。**SysEx でキャプチャ変数が存在する場合は必須**。CC / pitchBend 等の単一値イベントでは非表示（暗黙のソースのため） |
| `when` | 条件式。`64`（完全一致）/ `< 64` / `>= 64` / `0~63`（範囲）で入力する |
| `set` | 条件に一致したときにコンポーネント value にセットする値 |
| 行の順序 | 上から順に評価。ドラッグで並べ替えられる |
| `+ 条件を追加` | 新しい条件行を追加する |

#### set.expr エディター（右列 / `to`）

`to` 側のモードセレクターで `set.expr` を選択すると、右列に式入力欄を表示する。SysEx の複数キャプチャ変数を計算して1値にする場合に使う。

```
  右列
  ┌───────────────────────────────────────┐
  │ to: tempo.value                       │
  │ mode: [ set.expr      ▼ ]             │
  │                                       │
  │  expr: (hi << 2) | ((lo >> 5) & 0x03) │
  │                                       │
  │  使用可能な変数: lo, hi               │
  └───────────────────────────────────────┘
```

| 要素 | 内容 |
|---|---|
| `expr` 入力欄 | 式を直接入力。`set.expr` の構文に準拠 |
| 使用可能な変数 | `from.pattern` で定義されたキャプチャ変数名を一覧表示（参照用） |
| 構文チェック | ロード前にエラーをインラインで表示する |

#### raw events の追加 UI とサジェスト

追加・編集は手動入力が主体。入力フィールドにはサジェストを表示する。**サジェストの源泉はデバイスタイプごとに異なる。**

```
  [ noteOn ch1 note=_______________ ]
  ┌─────────────────────────────────┐
  │  noteOn ch1 note=60 vel=80      │
  │  noteOn ch1 note=62 vel=64      │
  │  noteOn ch1 note=64 vel=72      │
  └─────────────────────────────────┘
```

サジェストがない場合はドロップダウンを表示しない。

#### デバイスタイプごとのサジェスト設計

入力・編集 UI の構造は全タイプ共通。サジェストの源泉と取りうる値の表現はタイプごとに設計する。

| デバイスタイプ | direction | サジェストの源泉 |
|---|---|---|
| `midi` | input | 過去の手動入力値 / 編集中に受信した MIDI 信号 |
| `midi` | output | 過去の手動入力値のみ（送信済み信号の受け取りは不可） |
| `osc` | input | 過去の手動入力値 / 編集中に受信した OSC パケット |
| `osc` | output | 過去の手動入力値のみ |
| `osc-vrchat` | output | VRChat が自動生成するアバターパラメーター JSON（後述） |

出力側は送信済み信号を逆に受け取れないため、基本的にサジェストは過去の手動入力値のみになる。これは致し方ない。

#### `osc-vrchat` デバイスタイプ

VRChat はアップロード済みアバターの OSC パラメーター一覧を JSON ファイルとして自動生成する。これを読み込むことで出力側の raw events サジェストを提供できる。

| 要素 | 内容 |
|---|---|
| JSON の読み込み | プロファイル詳細設定画面でファイルパスを指定する。または VRChat のデフォルト出力先を自動検索する |
| サジェストの内容 | JSON に含まれるパラメーター名・型をサジェストとして一覧表示する |
| input 側への流用 | 同じ JSON を入力側（VRChat → Midori 方向）のサジェストにも使える場合は共用する |

`osc-vrchat` は `osc` の特殊タイプとして扱い、binding の構文は `osc` と共通とする。差分はサジェストの源泉のみ。

### レイアウトタブ

Preview / Monitor の描画構成（`layout` セクション）を定義する。

左ペインに definition パネルを置き、コンポーネントをキャンバスにドラッグ＆ドロップして並べる。definition 自体のインライン編集も同一画面で行える。

```
[ definition（左）]                   [ レイアウトキャンバス（右）]

  ┌──────────────────────┐            ┌────────────────────────────────┐
  │ + コンポーネントを追加 │            │  ████ ░ ████░░  upper  ╔═╗ ··│
  │                      │            │  ████ ░ ████░░  lower  ╔═╗ ··│
  │ ▼ upper  keyboard ⋮  │            │  ██░░           pedal  ╔═╗ ··│
  │     velocity  float  │            │ ┌─────────────────────────────┐│
  │     pressure  float  │──drag──►   │ │ ──●  upper_expression  ╔═╗ ││
  │     lateral   float  │            │ │ ○    upper_sustain     ╔═╗ ││
  │ ▼ lower  keyboard ⋮  │            │ └── グループ（row）  ╔═╗ ────┘│
  │     velocity  float  │            │                               │
  │ ▼ pedal  keyboard ⋮  │            │  （ここにドロップして追加）      │
  │     velocity  float  │            └────────────────────────────────┘
  │ ► upper_expression   │
  │   slider ⋮           │            選択中: グループ（row）
  │ ► upper_sustain      │            ┌────────────────────────────────┐
  │   switch ⋮           │            │ direction: row    ▼            │
  └──────────────────────┘            │ align:     start  ▼            │
                                      │ wrap:      □                   │
                                      │ color:     ─────────           │
                                      └────────────────────────────────┘
```

| 要素 | 内容 |
|---|---|
| definition パネル（左） | definition に定義されたコンポーネントの一覧。インライン編集可能。コンポーネントをキャンバスにドラッグして layout に追加する |
| レイアウトキャンバス（右上） | layout の構造をビジュアルにレンダリング。ドラッグで順番・親子関係を変更できる。キャンバス上でグループの追加・削除も行える |
| プロパティパネル（右下） | キャンバスで選択中のノード（コンポーネントまたはグループ）の `direction` / `align` / `wrap` / `color` を編集する |

#### definition パネルの操作

| 操作 | 内容 |
|---|---|
| キャンバスへドラッグ | layout への追加。同一コンポーネントを複数箇所に配置することはできない（1コンポーネント = 1エントリ） |
| `⋮` メニュー | コンポーネントの編集・削除 |
| `+ コンポーネントを追加` | 新しいコンポーネントを definition に追加する |

definition から削除したコンポーネントが layout に残っている場合は警告表示する（layout から自動除去はしない）。

#### キャンバスの操作

| 操作 | 内容 |
|---|---|
| ドラッグ＆ドロップ（definition → キャンバス） | layout に追加。ドロップ位置で挿入順が決まる |
| ドラッグ＆ドロップ（キャンバス内） | 順番・親子関係の変更 |
| グループに重ねてドロップ | そのグループの子として追加 |
| `╔═╗` ボタン | 選択してプロパティパネルを開く |
| グループ追加 | キャンバス上の空きエリアにドロップ、または右クリックメニューから「グループを追加」 |

#### keyboard コンポーネントの children 設定

`keyboard` コンポーネントを選択した場合、プロパティパネルに children の設定欄が追加表示される。

| 設定 | 内容 |
|---|---|
| 子コンポーネント | definition の value（`pressure` / `lateral` 等）に対して描画コンポーネント（`slider` / `pan`）を対応づける |

## 変換グラフ編集画面

デバイス構成を接続する変換グラフをノードグラフで定義する画面。

インタラクションは **Blender のシェーダーエディター** を参考にする。

```
Input Block（左）  →  計算ノード群（中央）  →  Output Block（右）
```

| 要素 | 内容 |
|---|---|
| Input Block（左端） | 入力 デバイス構成 の definition コンポーネント名をポートとして表示する |
| 計算ノード | キャンバス上に自由に配置・接続できる |
| Output Block（右端） | 出力 デバイス構成 の definition から ComponentState 構造をポートとして展開する。ポート一覧は選択済みの出力 デバイス構成 が確定した時点で自動生成される |
| 接続線 | ポート間をベジエ曲線で結ぶ。型不一致の接続は不可（視覚的にも示す） |

### ポートの型と Array 型

ポートは以下の型を持つ。型ごとにポートの色・形を変えて視覚的に区別する。

| 型 | 意味 | ポートの形（案） |
|---|---|---|
| `bool` | 真偽値 | ○（小円） |
| `float` | 浮動小数 | ◇（菱形） |
| `int` | 整数 | □（四角） |
| `pulser` | パルス（瞬間トリガー） | △（三角） |
| `bool[]` | bool の配列 | ○ に `[]` バッジ |
| `float[]` | float の配列 | ◇ に `[]` バッジ |

#### `{note}` と `*` の Array 表現

`{note}`（per-key 展開）と `*`（gather）はキャンバス上では **Array 型ポートとして1本の線** で表現する。88インスタンスを展開して並べるのではなく、配列を1つの値として扱う。

```
Input Block                         計算ノード
┌────────────────────┐             ┌───────────────────┐
│ upper.{note}.pressed   ○[] ─────►│ vel_bits_{note}   │
│ upper.{note}.velocity  ◇[] ─────►│  in  ◇[]          │
│                        │         │  bit_0 ○[]  ──────►│ Output Block
│                        │         │  bit_1 ○[]  ──────►│
└────────────────────┘             └───────────────────┘
```

- `{note}` サフィックスを持つノード ID（`vel_bits_{note}` 等）は Array 型を受け取り、Array 型を出力する per-element 処理ノードとしてキャンバス上に1つ表示する
- `*` gather も同様に `T[]` として1本の線で表現する
- Array 型と scalar 型の接続は型不一致として不可（視覚的にも示す）

#### Output Block のポート構造

出力 デバイス構成 の definition をそのまま ComponentState 構造に展開したものがポートになる。Input Block と対称な構造。

```
Output Block
┌──────────────────────────────┐
│ upper.{note}.pressed   ○[]   │  ← keyboard → Array 型
│ upper.{note}.velocity  ◇[]   │
│ upper.{note}.pressure  ◇[]   │
│ lower.{note}.pressed   ○[]   │
│ pedal.{note}.pressed   ○[]   │
│ upper_expression.value ◇     │  ← slider → scalar 型
│ upper_sustain.pressed  ○     │  ← switch → scalar 型
└──────────────────────────────┘
```

`keyboard` コンポーネントは `key_range` を持つため Array 型として展開される。`slider` / `button` 等は scalar 型のまま。入力が OSC で出力が MIDI の場合も同様に、出力 デバイス構成 の definition 構造がそのまま Output Block に現れる。

### インタラクション（Blender シェーダーエディター準拠）

| 操作 | 内容 |
|---|---|
| 中ボタンドラッグ / Space+左ドラッグ | キャンバスのパン |
| スクロール | ズームイン / アウト |
| `Shift+A` | ノード追加メニュー（検索付き）を開く |
| 左クリック | ノード / ポートの選択 |
| `G` | 選択ノードの移動（マウスに追従、クリックで確定・右クリックでキャンセル） |
| ポートからドラッグ | 接続線を引く。別ポートにドロップで接続、空きにドロップでキャンセル |

| `X` / `Delete` | 選択ノードの削除 |
| `Ctrl+Z` / `Ctrl+Shift+Z` | Undo / Redo |
| `Ctrl+A` | 全ノード選択 |
| ボックス選択（左ドラッグ） | 範囲内のノードを選択 |
| 右クリック | コンテキストメニュー |

- 保存 / 名前をつけて保存 / 開く

### 設定タブ

プロファイルのメタ情報とパイプライン構成を管理する。デバイス構成 を選択すると、その `driver` に応じた実デバイスの接続設定が展開される。

```
プロファイル名: [ ___________________________ ]

─── 入力 ───────────────────────────────────────
入力デバイス構成: [ yamaha-els03.yaml          ▼ ]
  driver: midi
  接続: [ ELS-03 Series                        ▼ ]  ← OS 認識済み MIDI デバイス一覧

─── 変換グラフ ────────────────────────────────────
変換グラフ: [ my-avatar.yaml                     ▼ ]

─── 出力 ───────────────────────────────────────
出力デバイス構成: [ vrchat-default.yaml         ▼ ]
  driver: osc-vrchat
  host: [ 127.0.0.1 ]       port: [ 9000 ]
  アバターパラメーター JSON: [ パスを選択...    ]  ← 任意
```

デバイス構成 を選択する前は接続設定欄を表示しない。

#### driver ごとの接続設定

| driver | 接続設定 |
|---|---|
| `midi` | OS が認識している MIDI デバイスの一覧から選択 |
| `osc` / `http` | host・port を入力 |
| `osc-vrchat` | host・port を入力 ＋ アバターパラメーター JSON のパス（任意） |

### ブリッジの状態表現

| 状態 | 実行ボタン | プレビュー・モニター |
|---|---|---|
| 停止中 | `[ ▶ 実行 ]` 有効 | 灰色で静的表示 |
| 起動中（中間） | `[ ◌ 起動中… ]` 非活性（スピナー表示） | 灰色のまま |
| 実行中 | `[ ■ 停止 ]` 有効 | 値変化に応じてコンポーネントが点灯 |
| 停止中（中間） | `[ ◌ 停止中… ]` 非活性 | 点灯が止まり灰色に戻っていく |

### イベントログの詳細

| 要素 | 内容 |
|---|---|
| 折りたたみヘッダー | クリックで展開 / 折りたたみをトグル。エラーが発生した場合は折りたたみ状態でもヘッダーを赤くして通知する |
| タイムスタンプ | `HH:mm:ss.SSS` 形式 |
| イベント種別 | `raw-event` / `device-state` / `signal` / `log` / `error` の5種。種別ごとに色分けする |
| ログ本文 | イベントの内容をテキストで表示 |
| クリアボタン | 表示中のログをすべて消去する（ブリッジの動作には影響しない） |
| 記録停止ボタン | クリックするとログの追記を一時停止する。再クリックで再開。ブリッジの動作には影響しない |
| 最大保持件数 | 直近 1000 件を保持。超過分は古い順に破棄する |

## エラー・空状態

### 初回起動

設定ファイルが存在しない状態でも通常通りダッシュボードを表示する。履歴が空なのでカード一覧が空になるだけ。特別なオンボーディング画面は設けない。

### プロファイルファイルの破損・バリデーションエラー

ダッシュボードからプロファイルを開こうとした際に読み込みエラーが発生した場合、エラーの内容を示すダイアログを表示してダッシュボードに戻る。

### デバイスが見つからない

プロファイルが要求するデバイスが OS 上に存在しない場合、プロファイル詳細画面の対応する入力プレビュー / 出力モニターを **灰色** で表示し、実行ボタンを無効化する。エラー（赤）ではなく「検証できないが定義はある」状態として扱う。

```
 ┌──────────────────┐
 │  upper keyboard  │  ← 灰色（デバイス未接続）
 │  ░░░░ ░ ░░░░░░  │
 │  MIDI デバイスが  │
 │  見つかりません   │
 └──────────────────┘

 [ ▶ 実行 ] ← 無効（グレーアウト）
```

デバイスが接続されると自動的に灰色が解除され、実行ボタンが有効になる。

### ランタイムエラー（実行中）

ブリッジ実行中にエラーが発生した場合、経路単位でエラーを伝播させ、該当コンポーネントを **赤** で表示する。イベントログのヘッダーも赤くなり、折りたたんでいても気づける。

## 用語の整理

| 用語 | 定義 |
|---|---|
| **デバイス構成** | raw events ↔ ComponentState の変換定義。1デバイス = 1ファイル（`devices/*.yaml`） |
| **変換グラフ** | ComponentState → Signal の変換ノードグラフ定義（`mappers/*.yaml`） |
| **プロファイル** | 入力デバイス構成・変換グラフ・出力デバイス構成・実デバイス接続設定を束ねた実行単位 |
