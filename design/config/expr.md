# 式言語仕様（set.expr）

バインディングの `to.set` において、複数のキャプチャ変数を用いた計算が必要な場合に使用する式言語の定義。

`set.expr` はドライバーに依存しない汎用の仕様。ただし、**式の中で参照できる変数はドライバーの物理型が「計算可能」と定義しているものに限られる**。ドライバーごとの計算可能性は各ドライバーの型定義を参照すること。

---

## 使用箇所

`set` フィールドにスカラー値やキャプチャ変数名ではなく `expr:` を指定することで、式を評価した結果を target に書き込む。

```yaml
- from:
    type: sysex
    pattern: [0xF0, 0x43, 0x70, 0x70, 0x40, 0x50, {lo}, {hi}, 0xF7]
  to:
    target: tempo.value
    set:
      expr: "(hi << 2) | ((lo >> 5) & 0x03)"
```

---

## スコープ

式の中で参照できる変数は、`from` で捕捉したキャプチャ変数のみ。

- 参照できるのはドライバーの型定義で **計算可能（✅）** とされた物理型のキャプチャ変数のみ
- 計算不可能な型（`byte[]`、`event` 等）のキャプチャ変数は式の中で使用できない
- 外部状態・コンポーネント値・グローバル変数は参照できない

---

## 構文

### リテラル

| 記法 | 例 | 説明 |
|---|---|---|
| 10 進数 | `0`, `127`, `240` | 符号なし整数 |
| 16 進数 | `0x00`, `0x7F`, `0xFF` | `0x` プレフィックス |

式文字列内のリテラルはブリッジのロード時にパースされ、内部の整数表現に変換される。ランタイムでの文字列パースは行わない。

### 変数

`from.pattern` の `{名前}` でキャプチャした変数名をそのまま参照する。

```
hi, lo, arg1, arg2, ...
```

### 演算子

| 種別 | 演算子 | 説明 |
|---|---|---|
| 算術 | `+` `-` `*` `/` `%` | 整数演算。`/` は切り捨て除算 |
| ビット | `&` `\|` `^` `~` | AND / OR / XOR / NOT |
| シフト | `<<` `>>` | 左シフト / 右論理シフト |
| グループ | `( )` | 評価順の制御 |
| 単項 | `-` `~` | 符号反転 / ビット反転 |

### 優先順位（高い順）

1. `( )` — グループ
2. `~` `-`（単項） — ビット反転、符号反転
3. `*` `/` `%` — 乗除
4. `+` `-` — 加減
5. `<<` `>>` — シフト
6. `&` — ビット AND
7. `^` — ビット XOR
8. `|` — ビット OR

---

## 評価結果の型

式の評価結果は整数。target フィールドの型（`float` / `bool` / `int`）に応じて自動変換される。

| target の型 | 変換 |
|---|---|
| `int` | そのまま代入 |
| `float` | target の `range` に対して線形正規化 |
| `bool` | `0` → false、それ以外 → true |

---

## 制約

- 条件分岐・ループは使用不可（`setMap` を使うこと）
- 関数呼び出しは使用不可
- 浮動小数点リテラルは使用不可（整数演算のみ）
- オーバーフロー・ゼロ除算の挙動は実装定義

---

## 使用例

### ELS-03 テンポのデコード

SysEx `F0 43 70 70 40 50 TL TH F7` から実テンポ値（40–280）を算出する。

```yaml
- from:
    type: sysex
    pattern: [0xF0, 0x43, 0x70, 0x70, 0x40, 0x50, {lo}, {hi}, 0xF7]
  to:
    target: tempo.value
    set:
      expr: "(hi << 2) | ((lo >> 5) & 0x03)"
```

### ビットマスクによる値抽出

```yaml
set:
  expr: "arg1 & 0x0F"   # 下位 4 bit を取り出す
```

### 複数バイトの合成

```yaml
set:
  expr: "(msb & 0x7F) << 7 | (lsb & 0x7F)"   # 14bit 値の合成
```
