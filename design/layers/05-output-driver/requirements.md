# Layer 5 — 出力ドライバー（Output Driver）要件

## 責務

メッセージの raw I/O 送信のみ。送信内容の意味解釈は行わない。

## インターフェース

```
入力: 出力ドライバー固有のメッセージ（Layer 4 出力）
出力: ネットワーク / シリアル等への送信
```

## 要件

| # | 要件 | 補足 |
|---|---|---|
| 1 | 意味解釈をしないこと | アドレスや値の内容を知らない。byte 列として送信する |
| 2 | 送信先はプロファイルの connection で定義された先のみに送信すること | 任意ホストへの送信を許可しない |
| 3 | 送信エラー時はログに記録してパイプラインを継続すること | 1パケットのドロップはリアルタイム性を優先して許容する |
| 4 | 複数送信先への並行送信に対応できること | プロファイルの `outputs` に複数エントリを持てる |
| 5 | ドライバーをインターフェース越しに差し替えられること | 上位層は具体的なドライバー実装を知らない |

## I/O モデル

ドライバーによって「どこへ / どうやって raw events を送り出すか」が異なる。上位層はこの違いを知らない。

| I/O モデル | 説明 | 例 |
|---|---|---|
| **ファイア＆フォーゲット** | 接続を持たずパケットを投げる | UDP（OSC） |
| **クライアント送信型** | 接続を確立してメッセージを送る | MIDI 出力, WebSocket クライアント |
| **HTTP クライアント型** | Signal 発生のたびに HTTP リクエストを送出する | HTTP POST/PUT |

HTTP の場合: Signal が発生するたびにプロファイルの connection で設定された URL へ JSON body を POST / PUT する。送出頻度が高い場合のスロットリングはドライバー内部の責務。

|---|---|---|
| UDP（`osc`） | ファイア＆フォーゲット | 初回実装 |
