# 画面構成と遷移

## 画面マップ

```
ワークスペース選択画面（アプリレベル）
└── ワークスペースを選ぶ ───→ ダッシュボード（ワークスペースレベル）
                                ├── プロファイル > 実行 ──────→ プロファイルトップ画面 ─→ 実行画面
                                ├── プロファイル > 編集 ──────→ プロファイルトップ画面
                                │                                    └── グラフを編集 ────→ 変換グラフ編集画面
                                ├── プロファイルを新規作成 ──→ プロファイルトップ画面
                                ├── 要素 > 編集 ─────────────→ デバイス構成編集画面
                                │                                    ├── バインディングタブ
                                │                                    ├── レイアウトタブ
                                │                                    └── プレビュータブ（デバッグ用）
                                ├── デバイスを作る ──────────→ デバイス構成編集画面
                                ├── プラグイン管理 ──────────→ Preferences 設定画面（プラグインタブ）
                                └── Preferences ─────────────→ Preferences 設定画面
                                                                     ├── 一般タブ
                                                                     ├── AI タブ
                                                                     └── プラグインタブ
```

## パンくずリスト

タイトルバー直下に常設する 1 行のパンくずリストでナビゲーションを提供する。

```
ワークスペース選択画面:
  Midori

ダッシュボード:
  [workspace-name]

デバイス構成編集画面:
  [workspace-name]  ›  yamaha-els03

プロファイル詳細画面:
  [workspace-name]  ›  My Profile

変換グラフ編集画面（プロファイルから遷移）:
  [workspace-name]  ›  My Profile  ›  変換グラフ

Preferences:
  [workspace-name]  ›  Preferences
```

各セグメントはクリックでその階層へ戻る。

| クリック対象 | 遷移先 |
|---|---|
| `Midori`（最左、常設） | ワークスペース選択画面 |
| `[workspace-name]` | ダッシュボード |
| 中間セグメント（例: `My Profile`） | 該当画面 |

`Midori` セグメントは全画面で常に表示し、ワークスペース切り替えの起点とする。

---

## ワークスペース切り替え

`Midori` セグメントをクリックするとワークスペース選択画面へ遷移する。現在のワークスペースを「閉じる」概念はなく、別のワークスペースを選ぶことで切り替わる。

ブリッジが実行中の場合は切り替え前に確認ダイアログを表示する：

```
ブリッジが実行中です。ワークスペースを切り替えるとブリッジを停止します。
[ キャンセル ]  [ 停止して切り替える ]
```

## 自動保存とナビゲーション

編集画面はすべて**逐次自動保存**とする。エディター内に「保存」「名前をつけて保存」ボタンは持たない。

| 画面 | 自動保存 |
|---|---|
| デバイス構成編集 | ✅ バインディング・レイアウトの変更をリアルタイムに保存 |
| 変換グラフ編集 | ✅ ノード・接続の変更をリアルタイムに保存（[→ `06-graph.md`](06-graph.md)） |
| プロファイル詳細（設定タブ） | ✅ 入力・出力・変換グラフの変更をリアルタイムに保存 |
| Preferences | ✅ 各設定項目の変更をリアルタイムに保存 |

自動保存のため、離脱時の「保存しますか？」ダイアログは発生しない。Undo / Redo（`Ctrl+Z` / `Ctrl+Shift+Z`）で変更を取り消せる。

タイトルバーのコンテキスト名に付く `●`（未保存インジケーター、[→ `02-tone.md`](02-tone.md)）は、自動保存がディスクへの書き込みを完了するまでの一瞬だけ表示する。

## ブリッジ実行中のナビゲーション

ブリッジが動作中でも画面遷移は制限しない。ブリッジはバックグラウンドで継続して動作する。

- タイトルバーのブリッジ状態インジケーター（[→ `02-tone.md`](02-tone.md)）が全画面で常時表示されるため、他の画面に移動してもブリッジの状態を確認できる
- ダッシュボードに戻ると、実行中プロファイルのカードに「● 実行中」バッジが表示される
- プロファイル詳細画面（プレビュータブ）に戻ることでブリッジを停止できる

## グローバル UI

### AI チャットパネル

全画面共通で画面右端に配置するフローティングパネル。詳細は [`08-ai.md`](../08-ai.md) を参照。

- 折りたたみ可能。折りたたみ時は右端にアイコンのみ表示
- 全画面から常にアクセス可能
- AI がファイルを保存・画面遷移した場合はパネル内にインジケーターを表示する

### タイトルバー

詳細は [`02-tone.md`](02-tone.md) を参照。

- 中央：編集中のコンテキスト名（プロファイル名・デバイス構成名等）。ダッシュボード表示中はワークスペース名
- 右：ブリッジの実行状態インジケーター
