# Timing / Tick 仕様

---

## 用語の定義

本ドキュメントで使う timing 関連の用語を明確に区別する。

| 用語 | コンテキスト | 意味 |
|---|---|---|
| **tick**（Midori） | Midori ランタイム | バックグラウンドスレッドの処理サイクル1回 |
| **tick**（MIDI） | MIDI プロトコル | PPQ（pulses per quarter note）単位の時間分解能 |
| **bps** | MIDI / テンポ | beats per second（= BPM ÷ 60） |
| **PPQ** | MIDI | 4分音符あたりのパルス数。時間分解能の単位 |

MIDI tick と Midori tick は別概念であり、Midori tick は実時間ベースで MIDI テンポに同期しない。MIDI タイミングの詳細は [MIDI タイミング](#midi-タイミング) 節を参照。

---

## Midori tick とは

バックグラウンドスレッドが**固定レートで実時間を刻む**処理サイクルを **1 tick** とする。

イベント受信とは独立して tick は常に進行する。入力ドライバーは受信したイベントをバッファに書き込み、tick スレッドが各 tick でそれを読み出して処理する。

```
[入力スレッド]  MIDI recv ──▶ state buffer に書き込み（非同期）

[tick スレッド] tick 0 ──┬── バッファ読み出し・コンポーネント値を更新
                         ├── マッパーのノードグラフ評価
                         └── 変化した値のみ出力ドライバーへ送信

               tick 1 ──┬── バッファ読み出し（変化なければスキップ）
                         ├── ...
```

---

## tick レート

tick レートはブリッジ全体で共通の1つの値を持つ。入力・出力ドライバーの種類に関わらず統一される。

要件：**MIDI イベントを取りこぼさず受け取れる精度**であれば具体値は実装に委ねる。MIDI の最小イベント間隔（約 320µs / 1バイト @ 31.25kbps）を下回らない tick レートであればよい。目安 1ms（1000Hz）。

---

## 出力の条件

tick ごとに**値が変化したコンポーネントのみ**出力ドライバーへ送信する。変化のない tick では出力は発生しない（ポーリングではなく差分出力）。

---

## tick と pulse

`pulse` は `bool` のサブタイプで、**発火した tick のみ `true`** となり、次の tick の開始時に自動的に `false` へリセットされる。

```
tick N:    pulse = true   ← 入力バッファから読み出し
tick N+1:  pulse = false  ← 自動リセット
tick N+2:  pulse = false
```

pulse が発火した tick 内では、そのノードグラフ全体の評価が `true` の状態で行われることが保証される。

---

## tick 内の評価順序

1 tick 内での処理順序:

1. **バッファ読み出し** — 入力スレッドが書き込んだイベントを取り出し、コンポーネントの値を更新する
2. **pulse リセット** — 前 tick に発火した pulse を `false` に戻す
3. **マッパー評価** — ノードグラフをトポロジカル順に評価する
4. **差分出力** — 変化した値を出力ドライバーへ送信する

---

## 入力スレッドと tick スレッドの協調

MIDI・OSC など各入力ドライバーはそれぞれ独立したスレッドで非同期にイベントを受信し、共有の state buffer に書き込む。tick スレッドは各 tick の先頭で buffer を drain してコンポーネント値を更新する。

```
[MIDI 入力スレッド]  ──▶ ┐
[OSC 入力スレッド]   ──▶ ├──▶ state buffer ──▶ [tick スレッド] ──▶ 出力ドライバー
[HTTP 入力スレッド]  ──▶ ┘
```

### state buffer の要件

- ロックフリーキューまたは言語ネイティブのチャンネル（channel / mpsc 等）で実装し、入力スレッドをブロックしない
- 複数ドライバーからの書き込みが安全に行えること
- tick スレッドは各 tick で buffer を全件 drain する（残留させない）

### tick 内に複数イベントが積まれた場合

同一 tick 内に複数イベントが到着していた場合、**FIFO 順に順次処理**する。

- 同一コンポーネント・同一フィールドへの複数書き込みは後勝ち
- sub-tick の時刻精度は失われる（1 tick 内のイベントは同時刻扱い）

## MIDI タイミング

MIDI プロトコルにおけるタイミングの定義。Midori tick とは独立した概念。

### MIDI tick（PPQ）

MIDI における時間分解能の単位。4分音符（quarter note）を何分割するかを **PPQ**（pulses per quarter note）で表す。

```
1 quarter note = PPQ ticks（MIDI）
```

一般的な PPQ 値: 24 / 48 / 96 / 480 / 960

### bps（beats per second）

```
bps = BPM ÷ 60
```

| BPM | bps |
|-----|-----|
| 60  | 1.0 |
| 120 | 2.0 |
| 180 | 3.0 |

### MIDI tick の実時間への変換

```
1 MIDI tick の長さ（秒）= 1 ÷ (bps × PPQ)
```

例: BPM=120、PPQ=96 の場合
```
bps = 120 ÷ 60 = 2.0
1 tick = 1 ÷ (2.0 × 96) = 約 5.2ms
```

### MIDI Clock（0xF8）

MIDI Clock メッセージは **24 PPQN 固定**で送信される。BPM にかかわらず4分音符あたり24回。

```
Clock 間隔（秒）= 1 ÷ (bps × 24)
```

例: BPM=120
```
Clock 間隔 = 1 ÷ (2.0 × 24) = 約 20.8ms
```
