# Preferences 設定画面

## メタファー

| サーフェス | メタファー |
|---|---|
| 画面全体 | **作業環境の設定パネル** — 作業内容ではなく作業環境を整える場所。一度設定したら触らない |
| 開発者カテゴリ | **テストベンチモードの切り替え** — 通常動作から開発・検証モードへ切り替えるスイッチ |

---

パンくずリストから `Preferences` に遷移する画面。**2カラムレイアウト**（左：カテゴリナビ、右：コンテンツ）。

```
┌──────────────┬────────────────────────────────────────┐
│  一般        │                                        │
│  AI          │  （選択中カテゴリのコンテンツ）          │
│  プラグイン   │                                        │
│  開発者      │                                        │
└──────────────┴────────────────────────────────────────┘
```

## 一般

```
─── 表示 ──────────────────────────────────────────────
テーマ:   ○ システム  ○ ライト  ● ダーク
言語:   [ 日本語  ▼ ]

─── 起動 ──────────────────────────────────────────────
[✓] 最後に開いたワークスペースを起動時に自動的に開く

─── 機能 ──────────────────────────────────────────────
[ ] AI を無効化
    チェック時、AI チャットパネルを非表示にし、AI 関連機能を無効にする

─── 更新 ──────────────────────────────────────────────
自動更新:
  ● 自動的にインストールする
  ○ 通知のみ（インストールは手動）
  ○ 無効
```

## AI

AI プロバイダーの接続設定。「AI を無効化」（一般 > 機能）が ON の場合はこのカテゴリ自体をグレーアウトする。

```
プロバイダー:
  ● Claude (Anthropic)
  ○ OpenAI
  ○ Ollama（ローカル）

── Claude ────────────────────────────────────────────
モデル:       [ claude-opus-4-6               ]
API キー:     [ ●●●●●●●●●●●●●●●● ]  [ 表示 ]
              API キーは keychain に保存されます

── 接続テスト ────────────────────────────────────────
              [ 接続テスト ]
              ✓ 接続成功  /  ✗ 接続できません（エラー詳細）
```

Ollama 選択時のみ追加フィールドを表示：

```
── Ollama ────────────────────────────────────────────
URL:          [ http://localhost:11434           ]
モデル:       [ llama3                           ]
              [ 接続テスト ]
```

## プラグイン

インストール済みプラグインの管理。

```
[ + プラグインを追加 ]

┌──────────────────────────────────────────────────────┐
│ Yamaha STAGEA ELS-03          v1.0.0  @yamaha-els03  │
│ デバイス構成: yamaha-els03.yaml                       │
│                                [ 更新確認 ]  [ 削除 ] │
└──────────────────────────────────────────────────────┘
```

プラグインカードに表示する内容：

| 要素 | 内容 |
|---|---|
| 表示名 | `plugin.yaml` の `display_name`（省略時は `name`） |
| バージョン | `plugin.yaml` の `version` |
| スコープ名 | `@<name>` 形式 |
| 提供内容 | デバイス構成 N 件 / ドライバー N 件 / デバイス種別 N 件（0 件の種別は省略） |
| 更新確認 | `git pull` を実行。最新の場合は「最新です」、更新がある場合は「更新する」に切り替わる |
| 削除 | 確認ダイアログを経てアンインストール |

### プラグイン追加フロー

```
[ + プラグインを追加 ] 押下
  ↓
URL 入力ダイアログ
  [ https://github.com/someone/midori-yamaha-els03 ]
  [ キャンセル ]  [ 次へ ]
  ↓
プレビュー（マニフェスト取得後）
  名前: Yamaha STAGEA ELS-03
  提供するデバイス構成: devices/yamaha-els03.yaml
  [ キャンセル ]  [ インストール ]
```

コードを含むプラグイン（ドライバーあり）の場合、プレビューに権限リストと警告を追加表示する（[→ `09-plugin.md`](../09-plugin.md)）。

空状態 → [`09-errors.md`](09-errors.md) を参照。

## 開発者

プラグイン・ドライバーの開発・検証を行う場合に使用する設定。通常ユーザーは触らない。

```
[ ] 開発モードを有効にする
    ワークスペース内の実行可能ファイルをドライバーとして認識し、
    開発者向けのログ出力を有効にします。
    信頼できるワークスペースでのみ使用してください。
```

開発モードが ON のとき、追加設定を展開表示する：

```
[✓] 開発モードを有効にする

  ─── ドライバー開発 ──────────────────────────────────
  [✓] ワークスペース内の実行可能ファイルをドライバーとして認識する
      .midori/workspace.yaml に宣言された drivers/ 以下のバイナリを
      インストール済みプラグインと同様に扱います

  ─── ログ ────────────────────────────────────────────
  [✓] 開発者ログを表示する
      ブリッジの詳細ログ・ドライバーの stdout/stderr をイベントログに出力します
```

開発モードを OFF に戻すと追加設定は折りたたまれ、設定値はリセットされる（次回 ON 時はデフォルト値に戻る）。
