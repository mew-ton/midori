# 変換グラフ編集画面

## メタファー

| サーフェス | メタファー |
|---|---|
| 画面全体 | **ブレッドボード** — IC チップと導線で回路を組み上げる作業台 |
| デバイスブロック | **DIP パッケージ IC** — 両側にピン（ポート）が並ぶ。折りたたみはチップを伏せる操作 |
| 変換ノード | **ロジックゲート・オペアンプ** — 入力→処理→出力が一体の小さな回路素子 |
| 接続線 | **ジャンパーワイヤー** — 被覆の色 = 信号の型。物理的に「つながっている」状態 |
| ポート形状 | **IC のピン形状** — 型ごとに形を変えることで、接続可否を物理的な嵌合として表現 |
| テスター | **オシロスコーププローブ / マルチメーター** — ワイヤーにクリップし、瞬時値と時間軸を統合したパネルで観測する |
| ノード配置レーン | **ブレッドボードの中央溝** — IC の脚を左右に分離する縦溝。UI に描画しないが、ノードの X スナップ先として機能する |
| ツールボックス | **パーツトレー / 部品ボード** — IC やコンデンサを整理しておく部品ケース。キャンバスへドラッグして使う |

---

デバイス構成を接続する変換グラフをノードグラフで定義する画面。

```
入力ブロック群（左）          計算ノード群（中央）          出力ブロック群（右）
デバイスごとに 1 ブロック                                   デバイスごとに 1 ブロック
縦に並ぶ・折りたたみ可能                                    縦に並ぶ・折りたたみ可能
```

| 要素 | 内容 |
|---|---|
| 入力ブロック群（左端） | 入力デバイス 1 つにつき 1 ブロック。ブロックのタイトルは `device_id`。縦に並ぶ |
| 計算ノード | 縦方向の暗黙レーンにスナップして配置。詳細は [レーン配置モデル](#レーン配置モデル) を参照 |
| 出力ブロック群（右端） | 出力デバイス 1 つにつき 1 ブロック。同様に縦に並ぶ |
| 接続線 | ポート間をベジエ曲線で結ぶ。スカラー型は実線、配列型は破線。型不一致の接続は不可 |

各ブロックはタイトル行クリックで折りたたみ可能。折りたたみ時はタイトルのみ表示し、接続線はポートの代わりにブロック端に集約する。

```
展開時:
┌─ ▼ yamaha-els03 ──┐                    ┌─ ▼ vrchat-default ┐
│ upper.{note}.pressed ○[]  ──────────►  │ upper_key.{note} ○[]
│ upper.{note}.vel    ◇[]                │ expression.value ◇
│ expression.value    ◇  ──────────────► └───────────────────┘
└────────────────────┘

折りたたみ時:
┌─ ► yamaha-els03 ──┐ ─────────────────► ┌─ ► vrchat-default ┐
└────────────────────┘                    └───────────────────┘
```

## ポートの型と視覚表現

型ごとに**形状**と**色**で識別する。色だけに依存しない（[→ 01-principles.md アクセシビリティ](01-principles.md)）。

### スカラー型

| 型 | 形状 | oklch |
|---|---|---|
| `bool` | ○（小円） | `oklch(62% 0.09 255)` |
| `float` | ◇（菱形） | `oklch(70% 0.10 75)` |
| `int` | □（四角） | `oklch(63% 0.09 195)` |
| `pulse` | △（三角） | `oklch(58% 0.09 305)` |

### 配列型：固定長と可変長は異なる型

**システム境界（デバイスブロックのポート）では配列は常に固定長（`T[N]`）。** 可変長配列（`T[]`）はグラフ内部のノード間でのみ出現しうる（`compact` ノードが生成する可変長配列がその例）。

**固定長配列**（`T[N]`）と**可変長配列**（`T[]`）は互換性がない別の型である。視覚的にも区別する。

| 型 | 形状 | 視覚表現 |
|---|---|---|
| `bool[N]`（固定長） | ○ の二重輪郭 | 外側の輪郭に N の数字バッジ |
| `float[N]`（固定長） | ◇ の二重輪郭 | 同上 |
| `int[N]`（固定長） | □ の二重輪郭 | 同上 |
| `bool[]`（可変長） | ○ の二重輪郭（破線） | バッジなし |
| `float[]`（可変長） | ◇ の二重輪郭（破線） | 同上 |
| `int[]`（可変長） | □ の二重輪郭（破線） | 同上 |

- **二重輪郭（実線）** = 固定長配列。サイズが定義から確定している
- **二重輪郭（破線）** = 可変長配列。サイズが実行時に決まる
- 接続線はスカラーが実線、配列型は破線（固定・可変共通）

### `{note}` と `*` の Array 表現

`{note}`（per-key 展開）は `T[N]`（固定長）、`*`（gather）は `T[]`（可変長）として表現する。キャンバス上では 1 本の線で表現する。

```
入力ブロック（yamaha-els03）              計算ノード
┌────────────────────┐                  ┌───────────────────┐
│ upper.{note}.pressed  ◎[88] ── ── ──►│ vel_bits_{note}   │
│ upper.{note}.velocity ⬦[88] ── ── ──►│  in  ⬦[N]         │
│                                       │  bit_0 ◎[N] ── ──►│ 出力ブロック
└────────────────────┘                  └───────────────────┘
```

（◎ = 二重輪郭の○、⬦ = 二重輪郭の◇）

### 出力ブロックのポート構造

```
出力ブロック（vrchat-default）
┌──────────────────────────────────┐
│ upper_key.{note}.pressed ◎[88]   │  ← keyboard → bool[88]
│ lower_key.{note}.pressed ◎[88]   │
│ expression.value         ◇       │  ← slider → float（scalar）
└──────────────────────────────────┘
```

## ノード・ワイヤーの状態表示

ノードとワイヤーは選択可能で状態を持つパーツであり、状態は**アウトライン**を共通形式として視覚化する。状態の種類に応じてアウトラインの色と挙動を切り替える。

| 状態 | 見せ方 | 発生する状況 |
|---|---|---|
| **選択** | 細めの白アウトライン | ユーザが対象をクリック / ボックス選択した |
| **エラー** | 目立つ赤アウトライン | 型不一致・違反接続・その他設計上の問題がある |
| **信号** | アニメーション付き緑アウトライン | テスター実行中、そのパスに信号が流れている |

複数状態が同時に成立した場合の優先順位は実装時に決定する（例：**エラー > 信号 > 選択**）。

### ワイヤーのアウトライン実装

ワイヤーは SVG の `path` で描画されるため、ネイティブの `outline` 属性を持たない。アウトライン風の表現は**同じパスを 2 本重ねる**ことで実現する：

- **背面**：太めのストローク（アウトライン色）
- **前面**：通常のストローク（ワイヤーの既定色）

背面ストロークが前面の両脇にはみ出し、アウトラインのように見える。状態変化は背面ストロークの色とアニメーション属性で表現する。

## テスター

**メタファー: オシロスコーププローブ / マルチメーター**

電子回路のテスターのように、グラフ上のワイヤーにプローブをクリップし、そのワイヤー上を流れる信号を**瞬時値と時間軸を統合したパネル**で観測する機能。入力から出力まで電気がきれいに流れているかを確認する。

### 動作条件

ブリッジが実行中のとき（またはテスト実行モード）にのみ有効。停止中はプローブ操作は行えない。

### プローブの取り付け

テスターモード中にワイヤーをクリックすると、そのワイヤーに**テスターパネル**が取り付けられる。パネルはワイヤー近傍に表示される。取り付け済みパネルをクリック（またはパネルの閉じる操作）で取り外し。

複数ワイヤーに同時に取り付けでき、各パネルは独立して動作する。ワイヤーが削除・付け替えされるとパネルも自動的に外れる。

**ホバーでは値を表示しない**（取り付け候補の affordance のみ）。瞬時値の観測は必ずパネル取り付け経由で行う。信号の有無だけを知りたい場合は [ノード・ワイヤーの状態表示](#ノードワイヤーの状態表示) の「信号」状態（緑アウトライン）で判別できる。

### テスターパネル

瞬時値と時間軸グラフを**一体で表示する専用ウィジェット**。ワイヤーが光るだけ・大きな数値が出るだけでは `int` / `float` / 配列型の値の構造を把握しきれないため、型に応じた表現を同居させる。

| エリア | 内容 |
|---|---|
| **リードアウト**（上部） | 現在値。遠目でも読める太字サイズ。楽器演奏中のちら見に耐える |
| **時間軸グラフ**（下部） | プローブ取り付け以降の値の変遷。X 軸は時間（直近 N 秒、N は設定または自動） |

リードアウトと時間軸グラフは型に応じて見せ方が変わる：

| 型 | リードアウト | 時間軸グラフ |
|---|---|---|
| `bool` | 大きな ON / OFF 表示（灯火表現） | 矩形波（ON/OFF の階段） |
| `int` | 大きな整数値 | 折れ線（ステップ変化） |
| `float` | 大きな小数値（桁数は自動） | 折れ線 |
| `pulse` | 最終発生タイミング（"0.3s ago" 等） | トリガーマーク（イベント発生点の縦線） |
| `T[N]`（配列） | 要素数と先頭数件 + 統計サマリ（最大/最小/平均） | 要素ごとの並列トレース or ヒートマップ（詳細は実装時） |

パネルはドラッグで**自由に配置**できる（重なりの自動整列はしない）。複数同時取り付け可能。**パネルの位置は保存されない**：グラフファイルに含めず、テスターモード内の一時的なデバッグ情報として扱う。取り付け直後の初期位置は当該ワイヤー近傍。

プローブ取り付け以降の値は継続して記録され、**ユーザが PC から離れて楽器を演奏し、戻ってきてから履歴を確認する**ユースケースに対応する。

### 信号パスのハイライト

ワイヤーを選択（またはプローブを取り付け）したとき、そのワイヤーが通る経路（上流ノード→接続線→下流ノード）を全てハイライトする。どこで信号が途切れているかを一目で確認できる。ハイライトは [ノード・ワイヤーの状態表示](#ノードワイヤーの状態表示) の「信号」状態（アニメーション付き緑アウトライン）を用いる。

```
  ◎[88]──── ── ──►  ⬦[N] ═══ ══ ══► ◇   ← ハイライト中の経路
              ↑ プローブ取り付け
         ┌────────────────────────┐
         │  0.76                  │ ← リードアウト
         │ ╱╲    ╱╲    ╱          │
         │╱  ╲__╱  ╲__╱           │ ← 時間軸グラフ
         └────────────────────────┘
```

### テスターモードの有効化

変換グラフ編集画面のツールバーに「テスト」ボタンを置く。押下するとデバッグ構成ダイアログを開き、使用するデバイス接続設定を確認・編集してからブリッジを内部的に起動し、テスターモードに入る。再押下または停止ボタンで終了する。

#### デバッグ構成ダイアログ

IDE のデバッグ構成に相当するUI。テスト実行のたびに開き、前回の設定を初期値として表示する。設定は preferences にキャッシュされ、次回起動時に自動補完される。

```
─── テスト構成 ─────────────────────────────────────
入力デバイス: [ ELS-03 Series              ▼ ]
出力デバイス: [ 127.0.0.1:9000（VRChat）   ▼ ]

              [ キャンセル ]  [ テスト開始 ]
```

#### ブリッジ同時起動の制限

デバイス構成プレビュー・変換グラフテスター・プロファイル実行のいずれかがブリッジを起動中の場合、別の起動操作はブロックされる。「別のブリッジが動作中です」とダイアログを表示し、現在の起動元画面へ誘導する。画面の導線上、同時起動が発生するケースは実質的に存在しない。

## ツールボックス

**メタファー: パーツトレー / 部品ボード**

電子工作の作業台には、IC やコンデンサなどの部品を整理しておく部品ケース（トレー）が脇にある。作り手はそこから必要な部品をつまみ上げてブレッドボードに刺す。UI でもこの動作をそのまま再現する。

利用可能なノードの一覧を表示するパネル。項目をキャンバスにドラッグ&ドロップすることで新規ノードを追加する。

### 配置

**未決定**。入力・出力デバイスの設定パネル（現状未定）との配置が揃うように、併せて決める。

### 構成

- **カテゴリ分け**：Signal / Math / Logic / ユーティリティ など。分類の粒度は実装時に調整
- **検索バー**：ノード名でフィルタ
- **折りたたみ**：カテゴリごとに展開 / 折りたたみ可能

**スペーサー**はユーティリティ（「その他」）カテゴリに配置する。通常ノードと同じ導線で扱えるため、特別な操作を覚える必要がない。

### インタラクション

ツールボックスの項目をキャンバスにドラッグすると、ドロップ位置で[レーン配置モデル](#レーン配置モデル)が発動する。既存ノード配置と同じルール（列スナップ / ガイド表示）が適用される。新規ノードは接続を持たないため列制約違反は発生せず、入出力ブロック列へのドロップのみ拒否される。

### レーン配置モデル

**メタファー: ブレッドボードの中央溝**

電子工作のブレッドボードには中央に縦溝があり、DIP パッケージ IC はこの溝を跨ぐように刺す。溝があることで IC の脚が左右に分離され、入力側と出力側の回路が物理的に決まる。同じ溝に沿って複数の IC が縦に並び、ブレッドボードを横に連結していくと信号は左→右に流れる。

この UI では、**溝そのものは描画せず、ノードの X スナップ先としてのみ機能させる**。見た目はフリーキャンバスのまま、X 座標だけが暗黙の縦レーンに吸着することで、左→右のデータフロー（[→ 01-principles.md](01-principles.md)）をノード配置そのもので構造的に強制する。

#### レーン構造

- **入力ブロック列**（最左）と **出力ブロック列**（最右）は固定
- **中間列**は動的に増減する
  - ノードを既存列間にドロップ → 新規列を挿入（右側の列は右シフト）
  - 列から最後のノードが抜ける → 列を自動削除（右側の列は左シフト）
- 列幅は**固定**
- 列内の Y 位置はノード数に応じた**固定 gap の縦 flex で自動計算**（自由座標は持たない）

ノードの位置情報は `(col_index, row_index)` のみ。`col_index` は入力列を 0 とし、出力列まで連番。`row_index` は各列内で 0 始まり。新規列作成時、最初のノードは `row_index = 0`。

#### ドラッグ中のフィードバック

ドラッグ中のみ、列境界を示す**縦の点線ガイド**が薄く表示される。何もしていないときはガイドは出ない。

| ドラッグ位置 | 挙動 |
|---|---|
| 既存列に近づく | 列内のノードが上下に退いて、挿入先 row のガイドが表示される |
| 列間の新規位置に近づく | 左右の列が左右に退いて、新規列のガイドが表示される |
| 同一列内で上下にドラッグ | 前後のノードが退いて row 入れ替え |

既存列への吸着と新規列挿入を切り替える距離閾値は、暫定値で実装し、触りながら調整する。

#### 配置制約と拒否演出

以下のドロップは拒否され、ノードは元位置に戻る：

- 移動先の col が、上流ノードのどれかの col 以下（左→右原則違反）
- 移動先の col が、下流ノードのどれかの col 以上（同上）

拒否対象位置でもガイドは表示するが**赤く**表示し、**カーソル付近に拒否理由の吹き出し**を出す。

#### `Alt` / `Shift+Alt` + ドラッグ：切断して配置

既定は拒否だが、モディファイアキーを併用することでワイヤーを切断しながら配置するモードに切り替わる。

| 修飾キー | 挙動 |
|---|---|
| `Alt` | 違反するワイヤーのみを切断して配置（最小破壊） |
| `Shift+Alt` | ドラッグ対象ノードの**全入出力ワイヤー**を切断して配置（ノードを完全に切り離して移動）。配置位置の制約は適用されない |

ドラッグ中のモディファイアキー候補は**モーダルヒント**として画面に常時表示される（位置は暫定でキャンバス隅など、ドラッグ終了と同時に消える）。例：

```
[Alt] 違反接続を切断   [Shift+Alt] 全接続を切断   [右クリック] キャンセル
```

修飾キー押下中は以下が共通して発生する：

- **切断対象ワイヤーが赤点滅**で予告表示される（ドロップ前に影響範囲が見える）
- カーソル付近の吹き出しが拒否理由から**「接続 N 本を切断して配置」**に切り替わる
- ドロップで切断 + 配置を**単一アクションとして確定**
- `Ctrl+Z` で切断ワイヤーとノード位置を**一括復元**

`Alt` の違反ワイヤーは移動後の col 関係で判定する：移動後に「上流ノード col ≥ 対象 col」または「下流ノード col ≤ 対象 col」となる接続が切断対象。`Shift+Alt` は違反/合法を問わず全入出力ワイヤーを切断するため、判定は不要で配置位置の制約もバイパスされる。

複数選択時は選択ノード全体の入出力ワイヤーを評価する。修飾キーを離すと切断予告は消え、通常の挙動（拒否または配置）に戻る。

#### スペーサー

列内に**空のスロット**を差し込むためのブランクノード。列スキップ接続のベジエが中間列のノードと重なるとき、スペーサーで本物のノードを押しやって配線が通る余白を作るために使う。

- **視覚**：点線ボーダーのみ（塗りなし）、ノード標準の高さを占める。ホバーでボーダーが濃くなる
- **ポート**：入出力とも持たず、ワイヤーの接続対象にならない。col 制約の判定にも寄与しない
- **追加**：[ツールボックス](#ツールボックス)のユーティリティカテゴリからドラッグ（通常ノードと同じ導線）。列内での row 位置は通常ノードと同様にドラッグで入れ替え可能
- **空列判定ではノードとして数えない**：本物のノードがすべて抜けた列は、残ったスペーサーごと自動削除される（配線専用レーンとして列を保持することはしない）

用途例：`col 2` に 1 つしかノードがなく、`col 1 → col 4` のスキップ配線と重なる場合、`col 2` の row 0 にスペーサーを挿入すると本物のノードが row 1 に押し下がり、row 0 側に配線が通る空間が生まれる。

### ワイヤー接続

#### スナップと識別

ワイヤードラッグ中、キャンバス上の要素は以下のように強調/減衰される：

- **強調**：接続可能な同型入力ポートを持つノード
- **減衰**：それ以外のノード、および既存のワイヤー（彩度を落として背景化）

ドラッグ先端は近接する入力ポートにスナップする。強調ノードへのスナップは通常表示、減衰ノードへのスナップも物理的には可能だが、その瞬間に：

- 掴んでいるノード（出力元）と引いているワイヤーが**赤く発光**
- カーソル付近にエラー理由の吹き出し（ノード配置拒否と同じ表示系で統一）

#### 禁止接続

- **同一列内接続**（左→右原則違反）
- **接続元より左側の列への接続**（同上）
- **型不一致の接続**

#### 列スキップ接続

`col 1 → col 4` のように中間列を跨ぐ接続は許可する。ベジエ曲線で直結描画する。間の列のノードと視覚的に重なる場合は、row を調整して避けるか、列内のノードが 1 つしかなく row 変更で解決できない場合は [スペーサー](#スペーサー)を挿入して本物のノードを押しやり、配線が通る空間を作る。

### ノードコンテキストメニュー

ノード上で右クリックするとコンテキストメニューが開く。項目：

- **つながっていない入出力を非表示**：未接続ポートを非表示化してノードをコンパクトに表示する。接続のあるポートのみが見えるようになる。再度選択で元に戻る
- **配列を展開 / 折りたたむ**：`T[N]` 型の配列ポート（入出力いずれも）を、N 個の個別要素ポートに展開する（または元の 1 ポートに戻す）。要素単位での接続が可能になる
- **複製**：同じ設定のノードを未接続状態で新規作成し、適切な位置に配置する
- **削除（線を補完）**：ノードを削除し、入出力ワイヤーを可能な範囲で上流→下流の直結に置き換える。型が合わない/対応関係が曖昧なワイヤーは切断される
- **削除（線を切断）**：ノードを削除し、接続されていたワイヤーをすべて切断する
- **名前をつける**：ノードの表示名をユーザ定義のラベルで上書きする（ヘッダーに表示）
- **入力の線をすべて切断** / **出力の線をすべて切断** / **すべての線を切断**：対象方向のワイヤーを一括切断する。「すべて」は `Shift+Alt` ドラッグの位置変更なし版として位置づく

すべての操作は `Ctrl+Z` で一括復元できる。

## 新規作成フロー

ダッシュボードから単体で新規作成した場合は、最初に使用デバイスを宣言するステップを挟む。

```
「変換グラフを新規作成」押下
  ↓
デバイス宣言ダイアログ
  入力デバイス: [ + 追加 ]
    @yamaha-els03/devices/yamaha-els03.yaml  id: yamaha-els03
  出力デバイス: [ + 追加 ]
    @vrchat-osc/devices/vrchat-osc.yaml      id: vrchat-osc
  [ キャンセル ]  [ 作成 ]
  ↓
ブロックが自動生成された状態でエディターを開く
```

プロファイル詳細画面の「マッピングを編集」から遷移した場合は、対応するプロファイルの `inputs` / `outputs` に基づいてブロックが自動生成された状態で開く。

宣言したデバイスは mapper の `input_devices` / `output_devices` に保存される。後から変換グラフエディター内で追加・変更もできる。
