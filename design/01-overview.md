# Midori 概要

> ステータス：設計フェーズ
> 最終更新：2026-04-03

## 概要

任意の入力ソース（初回は MIDI）の信号を受け取り、OSC シグナルに変換して送信するブリッジツール。演奏のリアルタイム同期（打鍵・エクスプレッション・左右ピッチ等）を主目的とし、VRChat アバター制御をプライマリターゲットとする。

ツールは **ブリッジ（Runtime）** と **GUI** に分離しており、ブリッジは設定ファイルを読んで動作するだけの自立したバイナリとして機能する。GUI はブリッジと疎結合で、モニタリングおよび設定編集を担う。

将来的には VRChat 向けに公開されているあらゆる OSC ツールの用途（心拍・顔トラッキング・キーボードホットキー・Twitch 連携等）をカバーする汎用パイプラインを目指す。

## ユースケース（初回）

- エレクトーンのアバターモデルを VRChat に持ち込み、演奏に合わせて打鍵・エクスプレッション・ペダルをリアルタイムに同期させる
- 楽器プロファイルをコミュニティで共有・配布する
- 他機種の MIDI 楽器ユーザーが自前のプロファイルを作成して利用する

## 配布モデル

```
コミュニティで公開・共有されるもの

  input-sources/*.yaml    楽器・デバイス1種 = 1ファイル。機種ごとの入力値を定義する
                          → 同じ楽器のユーザー全員が共用できる

プライベートで受け渡しされるもの（公開を前提としない）

  output-targets/*.yaml   VRChat にアップロードしたアバターのパラメーター構成に依存する
                          → アバターが異なれば内容も異なるため、個人〜グループ単位での共有

  mappers/*.yaml          入力から出力へのマッピング。ユーザー固有の調整を含む
                          → 個人ファイルとしての受け渡し想定

共有しないもの

  preferences.yaml        デバイス名・ホスト・ポートなど環境固有の値を含む
```

## セキュリティ要件

- ブリッジは明示的に渡された設定ファイルのみを読む
- ファイルシステムアクセスは preferences で許可されたパスに限定する
- ネットワーク送信は preferences の transports で定義された送信先のみ
- GUI の IPC は許可チャンネルのみを expose する
