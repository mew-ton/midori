# トンマナ

## 視覚的パーソナリティ

**モダン開発ツール（Raycast / Linear / Zed 系）**

Midori は特定ドメイン（音楽・MIDI 等）に紐づかない汎用シグナルルーティングツールである。ドメイン固有の「プロフェッショナルオーディオ感」は持たない。ニュートラルで飽きがなく、機能の密度を扱えるだけの情報密度設計を目指す。楽器奏者でも使えることはコピーとツールチップで担保し、UI 自体は汎用的に設計する。

---

## 基本方針

- **長さの単位はすべて `rem`**（base: 16px）
- **色の指定はすべて `oklch()`**（HSL・HEX は使用しない）
- OS のライト / ダーク設定に自動追従する。ユーザーは Preferences > 一般 で手動固定できる

---

## フォント

アプリにバンドルする。3 役割を 3 フォントでカバーする。

| 役割 | フォント | weight | 用途 |
|---|---|---|---|
| 装飾 | Geist | 700〜800 | アプリ名・大見出し |
| アプリテキスト | Geist | 400〜500 | ラベル・ボタン・説明文・一般 UI |
| プログラミング用途 | Geist Mono | 400〜500 | YAML・SysEx パターン・expr 式・イベントログ |
| 装飾（和文） | M PLUS 1 | 700〜800 | アプリ名・大見出し |
| アプリテキスト（和文） | M PLUS 1 | 400〜500 | ラベル・ボタン・説明文・一般 UI |
| プログラミング用途（和文フォールバック） | M PLUS 1 | 400 | 稀に日本語が現れるコード文脈 |

```css
--font-sans: "Geist", "M PLUS 1", sans-serif;
--font-mono: "Geist Mono", "M PLUS 1", monospace;
```

Geist / Geist Mono はバリアブルフォントのため weight を自由に使える。M PLUS 1 も Variable 版を使用する。和文のモノスペース専用フォントは設けない（SysEx・YAML キー・expr 式はほぼ ASCII のため）。

## カラーシステム

### 基本構造：モノクロベース + 彩度によるアクション層

UI 全体は**ニュートラルグレーのモノクロ基調**とし、色には彩度（oklch の C 値）で意味の強さを表現する。

```
C = 0（モノクロ）       C = 0.05〜0.10（muted）       C = 0.15〜0.22（vivid）
──────────────────    ──────────────────────────    ──────────────────────
背景・パネル            デバイスカード・ブロック          エラー
テキスト・アイコン       ポートの色・接続線               Submit ボタン
ボーダー               ノードカテゴリの背景              要注意の警告
                       → 「識別のための色」             → 「アクションを求める色」
```

**彩度が高いものが目に入ったら、何か対応が必要なサイン。** 識別色（デバイス・ノード等）は muted に留めることで、アクション要求色との混同を防ぐ。

### 状態カラー

| 変数 | oklch（ダーク） | 意味 |
|---|---|---|
| `--color-inactive` | `oklch(55% 0 0)` | 未接続・停止中 |
| `--color-error` | `oklch(55% 0.20 25)` | エラー（要対応） |
| `--color-active` | `oklch(65% 0.18 145)` | アクティブ（値が変化した瞬間） |

アクティブ状態（緑）は実行中のみ表示する。停止中・プレビュー時は使用しない。

### ポート型カラー（muted）

| 型 | oklch | 形状 |
|---|---|---|
| `bool` | `oklch(62% 0.09 255)` | ○ |
| `float` | `oklch(70% 0.10 75)` | ◇ |
| `int` | `oklch(63% 0.09 195)` | □ |
| `pulse` | `oklch(58% 0.09 305)` | △ |

固定長配列（`T[N]`）は同色＋二重輪郭（実線）＋ N バッジ。可変長配列（`T[]`）は同色＋二重輪郭（破線）。接続線はスカラーが実線、配列型が破線。詳細は [`06-graph.md`](../07-ui-ux/06-graph.md) を参照。

### デバイスアクセントカラー（8 色パレット）

プロファイルに追加されたデバイスに自動で割り当てる。同一プロファイル内でユニークになることを保証する。ユーザーはレイアウト設定で変更できる。

| 名前 | oklch |
|---|---|
| muted-coral | `oklch(65% 0.09 22)` |
| muted-rose | `oklch(63% 0.09 355)` |
| muted-indigo | `oklch(58% 0.09 270)` |
| muted-lime | `oklch(66% 0.09 130)` |
| muted-cyan | `oklch(65% 0.08 200)` |
| muted-slate | `oklch(58% 0.07 245)` |
| muted-sand | `oklch(70% 0.08 80)` |
| muted-mauve | `oklch(60% 0.09 315)` |

赤・緑はエラー・アクティブ状態と混同するリスクがあるため除外。ポート型色（blue / amber / teal / purple）と被らない色域から選定。

### ノードカテゴリの背景 tint（ダーク）

| カテゴリ | tint oklch | アイコン例 |
|---|---|---|
| 単純変換（scale / clamp 等） | なし | ⇄ |
| 算術・論理（add / and 等） | `oklch(20% 0.03 75)` | ± |
| 配列操作（flatten / collect 等） | `oklch(20% 0.03 195)` | [] |
| 信号・時間（latch / smooth 等） | `oklch(20% 0.03 305)` | ⏱ |
| 複合（gate / if 等） | `oklch(20% 0.03 255)` | ⊕ |

## サーフェス階層（ダークテーマ基準）

デザインの起点はダークテーマとする。ライトテーマは同一の構造で明度を反転させる。

| レベル | oklch | 用途 |
|---|---|---|
| `--surface-0` | `oklch(12% 0.005 250)` | アプリ背景 |
| `--surface-1` | `oklch(16% 0.005 250)` | パネル・スプリットペイン |
| `--surface-2` | `oklch(20% 0.005 250)` | カード・ノード |
| `--surface-3` | `oklch(25% 0.005 250)` | ホバー・インプット・選択中 |

## カスタムタイトルバー

Electron のカスタムタイトルバーを使用する。

```
macOS:
[ ● ○ ○ ]    Yamaha ELS-03 ●                [ ◌ 起動中… ]
  traffic      編集中のコンテキスト名＋未保存     ブリッジ状態
  lights（左）  （中央寄せ）                     （右）

Windows:
[ 🌿 Midori ]  Yamaha ELS-03 ●  [ ◌ 起動中… ]  [ — □ × ]
```

コンテキスト名はファイルパスではなく**編集対象の表示名**（デバイス構成名・プロファイル名等）を使う。ダッシュボード表示中は「Midori」のみ。

- macOS: `titleBarStyle: 'hiddenInset'` で traffic lights を残しつつカスタム領域を確保する
- ブリッジの実行状態をタイトルバーに常駐させ、他ウィンドウに隠れていても状態を確認できるようにする
- ドラッグ領域: `-webkit-app-region: drag`。ボタン・インタラクティブ要素は `no-drag`
